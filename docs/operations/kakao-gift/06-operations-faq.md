## 1. 앱 에러 메시지별 조치

사용자가 쿠폰 등록 시 앱/웹에 뜨는 토스트 메시지입니다. **메시지 문구로 찾아** 분류와 조치를 확인하세요.

### 1-1. 빠른 조회표

| 사용자가 보는 메시지 | 코드 | 분류 | 1차 조치 |
|----------------------|------|------|----------|
| "앱을 최신 버전으로 업데이트 해주세요" | CO012 | 🟦 사용자 안내 | 앱 최신 업데이트 후 재시도 안내 |
| "쿠폰을 찾을 수 없습니다" | CM001 | 🟦 사용자 안내 | 쿠폰번호 재확인(공백·자릿수) |
| "만료된 쿠폰입니다" | CM002 | 🟦 사용자 안내 | 유효기간 만료 — 연장·회수 불가 안내 |
| "이미 사용된 쿠폰입니다" | CM003 | 🟦 사용자 안내 | 사용 이력 확인 후 안내 |
| "상품을 찾을 수 없습니다" | CM004 | 🟥 운영 점검 | **상품 등록/활성 여부 확인** |
| "일시적인 서비스 오류가 발생했습니다" | CM005 | 🟨 일시 오류 | 재시도 안내 → 지속 시 쿠프마케팅 |
| "일시적인 통신 오류가 발생했습니다" | CM006 | 🟨 일시 오류 | 재시도 안내(쿠폰 원복됨) |
| "하트 충전에 실패했습니다" | CM007 | 🟨 일시 오류 | 재시도 안내 → 반복 시 개발 |
| "스킬 이용권 발급에 실패했습니다" | CM008 | 🟨 일시 오류 | 재시도 안내 → 반복 시 개발 |
| "쿠폰 정보를 찾을 수 없습니다" | CM009 | 🟥 시스템 결함 | **즉시 개발 문의**(상품 정합성) |
| "결제가 취소된 쿠폰입니다" | CM010 | 🟦 사용자 안내 | 카카오 결제 취소 상태 안내 |

> **분류 의미**
> - 🟦 **사용자 안내** — 시스템 정상. 사용자에게 상황만 안내하면 끝.
> - 🟨 **일시 오류** — 보통 재시도로 해결. 반복되면 개발/쿠프마케팅 확인.
> - 🟥 **운영 점검 / 시스템 결함** — 우리 쪽 조치 필요. CM004는 운영자가 상품 점검, CM009는 즉시 개발 에스컬레이션.

### 1-2. 코드별 상세 조치

#### CO012 — "앱을 최신 버전으로 업데이트 해주세요" 🟦
- **의미**: 구버전 앱에서 카카오 쿠폰(`90`/`91`)을 입력. 구버전은 신규 등록 경로를 몰라 가드에 걸림(정상 동작).
- **조치**: 앱스토어에서 **최신 버전 업데이트 후 재시도** 안내. 결함 아님.

#### CM001 — "쿠폰을 찾을 수 없습니다" 🟦
- **의미**: 쿠프마케팅 조회(L0)에서 유효하지 않은 쿠폰번호.
- **조치**: 쿠폰번호 **오타·공백·자릿수** 재확인. 카카오 구매 내역의 번호와 대조. 번호가 맞는데도 반복되면 → 쿠프마케팅에 쿠폰 상태 문의.

#### CM002 — "만료된 쿠폰입니다" 🟦
- **의미**: 쿠폰 유효기간 경과(L0 `8003` 또는 만료일 < 오늘).
- **조치**: 만료 정책 안내. 연장·재발급·회수는 **불가**(카카오/쿠프 정책). 환불 문의는 카카오 채널로 안내.

#### CM003 — "이미 사용된 쿠폰입니다" 🟦
- **의미**: 이미 사용 처리된 쿠폰(L0 `8005` 또는 사용 플래그 Y).
- **조치**: `coop_marketing_coupon_usage` 에서 해당 `coupon_code` 사용 이력 확인 — 본인이 이미 등록했는지 안내. 사용한 적 없다고 주장하면 → `coop_marketing_api_log`(L0/L1) 확인 + 쿠프마케팅 대조(중복 사용/오인 여부).

#### CM004 — "상품을 찾을 수 없습니다" 🟥 (운영 점검)
- **의미**: 쿠폰의 상품코드(ProductCode)에 매핑된 상품이 없거나 **비활성**.
- **조치 (운영자)**:
  1. `coop_marketing_product` 에 해당 `product_code` **등록 여부** 확인 → 누락이면 [01 상품 등록](01-product-registration.md)으로 등록.
  2. 등록돼 있으면 `is_active` 확인 → 의도치 않게 `false` 면 활성화.
  3. 신규 출시 직후 다발하면 **상품 등록 누락 신호** — 출시 체크리스트 점검.

#### CM005 — "일시적인 서비스 오류가 발생했습니다" 🟨
- **의미**: 쿠프마케팅 사용승인(L1) 응답 오류.
- **조치**: 잠시 후 재시도 안내. `coop_marketing_api_log`(L1 `response_data`) 확인. 특정 시간대 집중·지속되면 → 쿠프마케팅 문의.

#### CM006 — "일시적인 통신 오류가 발생했습니다" 🟨
- **의미**: 쿠프마케팅 API 타임아웃/네트워크 오류.
- **조치**: **쿠폰은 소진되지 않음**(망취소 L3로 원복) → 그대로 재시도 안내. 광범위하게 발생하면 → 쿠프마케팅 API 상태·서버 인프라 확인(개발/`/dev-infra`).

#### CM007 — "하트 충전에 실패했습니다" 🟨
- **의미**: L1 성공 후 하트 충전 단계 실패 → **자동으로 쿠프마케팅 취소(L2)** 되어 쿠폰 원복됨.
- **조치**: 재시도 안내(쿠폰 재사용 가능). 반복되면 → 개발 문의(HeartService 오류 로그 + winston).

#### CM008 — "스킬 이용권 발급에 실패했습니다" 🟨
- **의미**: L1 성공 후 100% 할인 쿠폰 발급 실패 → **자동 L2 취소** 되어 쿠폰 원복됨.
- **조치**: 재시도 안내. 반복되면 → 개발 문의 + 해당 스킬 상품의 `coupon_spec_seq` 정합성 확인([01 §5-B 진단 SQL](01-product-registration.md#5-등록-후-검증)).

#### CM009 — "쿠폰 정보를 찾을 수 없습니다" 🟥 (시스템 결함)
- **의미**: 스킬 상품의 `coupon_spec_seq` 에 해당하는 CouponSpec 이 없음. **상품↔쿠폰 정합성 결함**(ISS-071 의 G/D/H 케이스 — 좀비 상품·타입 전환·부분 실패).
- **조치**: **즉시 개발 에스컬레이션.** 동시에 운영자가 [01 §5-B 진단 SQL](01-product-registration.md#5-등록-후-검증)로 좀비 상품 여부 확인. 해당 상품은 사용 불가 상태이므로, 정합화 전까지 사용자 추가 발생 차단(비활성 검토).

#### CM010 — "결제가 취소된 쿠폰입니다" 🟦
- **의미**: 카카오 측에서 결제가 취소된 쿠폰(L0 `8099`).
- **조치**: 카카오 구매/결제 취소 상태 안내. 헬로우봇에서 복구 불가 — 카카오 채널로 안내.

## 2. 조치 공통 — 어디서 확인하나

| 확인 대상 | 위치 | 무엇을 보나 |
|----------|------|------------|
| 쿠폰 사용/취소 상태 | `coop_marketing_coupon_usage` | `status`(used/canceled), `canceled_at`, `recovered_at`, `user_seq` |
| 상품 등록·활성 | `coop_marketing_product` | `product_code`, `is_active`, `product_type`, `coupon_spec_seq` |
| 쿠프마케팅 API 전문 | `coop_marketing_api_log` | `process_type`(L0~L3), `request_data`, `response_data` |

## 4. 기타 운영 FAQ

> 운영하며 반복되는 질문을 누적합니다. (초기 항목 — 추후 보강)

**Q. 사용자가 "쿠폰을 등록했는데 하트가 안 들어왔어요" 라고 합니다.**
A. `coop_marketing_coupon_usage` 에서 해당 사용자/쿠폰의 `status` 확인.
- `used` 이고 `heart_log_seq` 있음 → 충전 완료. 하트 잔액·사용 내역 안내(이미 소비했을 수 있음).
- `used` 인데 `heart_log_seq` 없음 → 지급 누락 의심 → 개발 문의(2단계 커밋 중 중단 가능성).
- `canceled` → 취소된 쿠폰. 취소 경위 확인([02 결제·취소](02-payment-cancellation.md)).

**Q. 같은 쿠폰을 다시 등록하면 어떻게 되나요?**
A. 정상 사용 쿠폰은 재등록 시 "이미 사용된 쿠폰입니다"(CM003). 운영자가 취소한 쿠폰은 재사용 가능(UPSERT로 갱신).

**Q. 상품을 잘못 등록했는데 어떻게 고치나요?**
A. 스킬 교환권은 **수정 대신 비활성 + 신규 등록**이 원칙입니다(특히 `fixed_menu_seq`). 이유와 케이스: [01 §4 수정·삭제 시 주의](01-product-registration.md#4--수정삭제-시-주의-iss-071).
