# YoY 구매자수·매출 하락 원인 — 5세 연령 × 플랫폼 세그먼트 진단

> 핵심 KPI(구매자수·매출)의 YoY 하락을 **세그먼트 단위로 분해**하여 어느 (연령 × 플랫폼) 셀이 하락을 견인하는지 진단하는 분석 프로젝트.

**관련 TODO**: [TODO-021](../../todos/TODO-021-yoy-decline-segment-analysis.md)

## Problem

서비스 구매자수·매출이 YoY 하락 추세로 의심되지만, **하락이 전체적으로 균등한지 / 특정 세그먼트에 집중되는지** 가시화되지 않은 상태. 균등 하락이면 거시적 원인(시즌·트렌드·요금 정책), 특정 세그먼트 집중이면 미시적 원인(해당 연령대 콘텐츠 fit·플랫폼 결제 funnel 문제·플랫폼별 마케팅 갭) 으로 진단이 갈림.

## Customer Job (분석 의뢰자 관점)

1. **하락 원인의 위치 식별** — 5세 연령 버킷 × 플랫폼(iOS/Android/Web) 매트릭스에서 하락 기여도 큰 셀 확인
2. **균등 하락 vs 집중 하락 판정** — 어떤 액션이 효과적일지 정책 결정 근거
3. **신규 vs 기존 사용자 분해** (스코프 확정 시) — 신규 유입 감소 vs 기존 이탈 어느 쪽이 더 큰지

## Requirement (스코프 합의 전 — 초안)

> ⚠ 아래는 등록 시점 초안. **첫 `/dev-data` 분석 세션 진입 전에 사용자와 픽스 필요**.

- **R1. 핵심 지표 YoY 비교** — 구매자수·매출의 전년 동기 대비 변동률 (월 단위 + 전체 누적)
- **R2. 5세 연령 × 플랫폼 매트릭스 분해** — R1 의 변동률을 (`age_group_5yr` × `platform`) 셀로 분해, 셀별 기여도 산출
- **R3. 하락 셀 진단** — 상위 N개 하락 기여 셀에 대해 결제 funnel 단계별 분해 (방문 → 구매 시도 → 결제 완료)

## 합의 필요 사항 (분석 착수 전)

| 항목 | 옵션 | 비고 |
|---|---|---|
| YoY 비교 기간 | A. 동월 (2026-04 vs 2025-04) / B. 직전 N개월 누적 / C. 분기 | 시각화·신호 안정성 트레이드오프 |
| 매출 정의 | A. 총 결제액 KRW / B. IAP 만 / C. IAP + 외부 쿠폰(카카오) | 카카오 선물하기는 2026-04 출시 → 외부 비교 시 별도 처리 |
| 구매자수 정의 | A. 결제 trx 발생 unique user / B. 첫 결제자 / C. 재결제자 / D. 모두 분리 | 신규/기존 분해 가능 여부 |
| 플랫폼 분류 | A. Firebase `event_platform` / B. 서버 origin / C. 결제 채널(Google/Apple/Kakao) | 동일 user의 다중 플랫폼 결제 케이스 정의 필요 |
| 산출 형태 | A. Looker 대시보드 / B. 일회성 BQ 리포트(노션) / C. 양쪽 | 반복 모니터링 의도 여부 |

## 사전 가설 (검증 대상)

- H1. 하락은 **특정 연령 버킷에 집중** 되어 있다 (예: 21-25 / 26-30 등 핵심 결제층).
- H2. 하락은 **플랫폼별로 차등** 되어 있다 (Google Play 결제 정책 변화·iOS 가격 정책 영향).
- H3. **신규 유입 감소**가 기존 이탈보다 큰 기여를 한다 (신규 결제자 YoY 감소율 > 재결제자).
- H4. 카카오 선물하기 출시(2026-04~) 가 외부 채널 분담을 만들었지만 **자사 IAP 하락을 상쇄하지 못한다**.

가설은 분석 결과로 확정/기각.
