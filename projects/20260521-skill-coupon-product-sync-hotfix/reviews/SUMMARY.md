## D-1~D-5 권장안 재평가 (REVIEW-5 결론)

| 결정 | 1차 권장 (requirements.md §8) | 통합 리뷰 결론 |
|------|-------------------------------|---------------|
| D-1 (delete cleanup) | soft | 1차 권장은 유효하나 발급분의 "표시 필드 보호" 정책 (D-6) 과 묶어 재정의 필요 |
| D-2 (productType 전환) | 비허용 | 유효 — 단, **P1 핫픽스로 묶지 말고 1~2 라인 validation 만 추가하여 P3 로 강등** (R5 F-3) |
| D-3 (name UNIQUE) | suffix `${productName} 이용권 #${productCode}` | **재설계 필수** — 5/5 리뷰어 지적. D-15(노출 경로)·OPS-2(productCode 소유자)·발급분 표시 보호(D-6) 모두 선결 |
| D-4 (정합화) | 위반 발견 시 일회성 SQL | **결정 이연** — 진단 SQL 실행(T-1~T-4) 결과 후 (D-14 마감 결정 후) |

## 권장 다음 단계

본 통합 리뷰 결과를 사용자에게 보고 후, 다음 순서로 진행:

### 1. 즉시 (코드 수정 없이)

- **진단 SQL T-1~T-4 작성·실행** (5/22 출시 직후, D+3 마감 목표) — D-4·D-5·일정 결정의 선행 입력 (C-7, D-14)

### 2. requirements.md 보강 (코드 수정 없이)

진단 SQL 실행과 병행 가능. 본 통합 리뷰 결과 반영:

- **§7 검증 기준** 보강: 측정 도구·주기·책임자 명시 (R3 F-10)
- **§3 FR** 보강: 동시성·격리 수준 명시 (C-6), 결함 E/H/I 매핑 명시 (C-4), FR-1.5↔FR-4.1 충돌 해소 (C-5)
- **§5 제약** 보강: 롤백·점진 배포 정책 (C-8)
- **§8 결정 보류** 확장: D-1~D-5 → D-1~D-22 + OPS-1~9 + D-A~F. 트레이드오프·대안 비교 보강
- **§2 영향 범위** 보강: 신규 테이블·컬럼 발생 시 갱신 책임 명시 (R3 F-12)
- **신규 §**: 결함 매핑 검증 표 (REVIEW-3 의 매핑 표 참고)
- **신규 §**: 발급 쿠폰 스냅샷 정책 (D-6 결정 후 본문화)

## 결론 — 한 줄

요구사항 초안은 **방향은 옳지만 본문이 비어 있는 영역(SOP, 트랜잭션, 동시성, 정량화, 스냅샷, 매핑)이 너무 많아 /architect 가 그대로 받으면 임의 결정의 누적으로 부채가 커진다**. 진단 SQL 실행 + 본 통합 리뷰의 C-1~C-8 보강 + D-1~D-17 협의 후에야 다음 단계 진입 가능.
