재무 데이터에 따르면:

| 질문 | 실제 데이터 |
|---|---|

### 결정적 증거: 구매자당 구매건수 안정성

"월간 구매자당 구매건수" 그래프는 **1.92~2.16의 매우 협소한 범위**를 보이며 3/22 전후로 어떤 구조적 변화도 없다. 계절성(12월 피크, 1월 저점)만 존재.

이 수치의 의미:
```
총 구매건수    = 구매자 수 × 구매자당 구매건수
구매자 비율    = 구매자 수 / MAU
```

구매자당 구매건수가 안정적이라면, 총 구매건수의 변화 = 순수하게 구매자 수 변화.  
그런데 일별/월별 총 구매건수도 3/22에 급변이 없다 → **실제 구매자 수는 3/22에 급감하지 않았다.**

### 2순위 가설 기각: MAU 급증 없었음

MAU 그래프(Graph 1)가 "MAU 급증으로 인한 비율 희석" 가설을 직접 반박한다.
- MAU는 급증이 아닌 **지속적 하락** (55k → 35k, 7개월 간 -36%)
- 분모(MAU) 증가로 비율이 희석되는 시나리오는 데이터와 정반대

| 순위 | 가설 | 검증 결과 |
|------|------|----------|
| **1순위** | **앱 이전(2/10) 통계 지연 분리** | **✅ 확정** — 실제 구매 행동 불변, 지표 계산 기준 변화로 정상화 |
| 2순위 | MAU 급증 → 비율 희석 | ❌ **기각** — MAU는 급증이 아닌 하락 |
| 3순위 | Google 참여도 측정항목 개선 아티팩트 | ❌ **기각** — 공식 적용일 4/22, 3/22와 무관 |

**확정된 메커니즘:**
```
2/10 앱 이전 직후: 신계정에 "얼리 마이그레이터" (고관여 사용자)만 포함
                  → Play Console 구매자 비율 인위적으로 높음 (13.11%)

~3/22 (~40일 후): 전체 유저풀이 신계정 월간 집계에 포함
                  → 저관여 포함 전체 평균으로 정상화
                  → 6.63% = 실제 기준값

이후 1년+ 유지:   신계정에서 새로 쌓이는 데이터가 실제 서비스 수준을 반영
                  → 6~8%가 정상 범위
```

**경영적 함의:**
- 이것은 **비즈니스 문제가 아니다** — 실제 구매자와 매출은 변하지 않았다
- **6.6%가 실제 기준값**이며, 13.11%는 앱 이전 초기의 선택 편향 수치
- 단, MAU와 구매건수가 함께 하락하는 트렌드(55k→35k, 14k→9k)는 별도 사업적 관심 필요
