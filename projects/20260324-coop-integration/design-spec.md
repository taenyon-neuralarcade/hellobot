#### 로딩 상태 규범 (ISS-042, 2026-04-22 확정)

- **원칙**: 등록 버튼 탭 → API 응답 수신 전까지 **즉시 `disabled` + 회색 톤으로 변경**. 성공·실패 응답 수신 시점에 활성 복귀.
- **스피너 금지**: 버튼 내부 스피너, `CircularProgressIndicator`, GIF, 풀스크린 오버레이 인디케이터 등 별도의 로딩 애니메이션 요소를 **신규로 도입하지 않음**. 중복 요청 방지는 `disabled` 단독으로 충족.
- **색상 토큰**: 기존 "비활성 (입력 없음)" 상태와 동일한 `#C6C8CC`(GRAY 400) 배경 + `#EDEDEE`(GRAY 200) 텍스트 토큰을 **재사용**. 신규 "로딩 전용 회색 토큰"을 만들지 않음.
- **레거시 예외 (문서상 명시)**: Web(`<Loading />` 풀스크린 스피너) / Android(`CircularProgressIndicator` 버튼 내 인라인 스피너)는 본 규범 확정 이전에 이미 스피너 포함 구현이 병합되어 배포 단계에 있음. 회귀 리스크를 이유로 **롤백하지 않음** (2026-04-22 사용자 확정). 신규 구현(iOS 등)과 차후 리팩터링은 본 규범("disable + 회색")을 따른다.

#### 레이아웃 일관성 규범 (ISS-031, 2026-04-22 확정 · 갱신)

스킬 이용권 카드는 **동일 리스트에 병치되는 일반 쿠폰 카드(`CouponItem` / `CouponItemCell` 등)와 박스 높이·점선 위아래 여백·내부 수직 리듬이 완전히 일치**해야 한다. 구현 편차를 줄이기 위해 다음 원칙을 명시한다.

- **"같은 모양에 요소만 미노출" 원칙**: 스킬 이용권에 없는 행(서브텍스트 "N원 이상 결제 시", 만료일 등)은 **해당 행의 레이아웃 공간은 유지(reserve)하고 내용만 숨긴다**. iOS 관점에서는 `hidden = true`만 적용하고 `flex.isIncludedInLayout`/높이 제약은 유지. 웹/Android 관점에서는 `visibility: invisible`(iOS의 `alpha 0` 상당) 또는 동일 높이의 빈 placeholder 뷰를 배치한다. 행을 제거(collapse)하거나 상위 컨테이너 `gap`만으로 간격을 구성하지 않는다.
- **측정값 고정 금지**: 스킬 이용권 카드 높이·점선 간격·서브텍스트 영역 높이 등 **새로운 고정 픽셀 값을 지정하지 않는다**. 각 플랫폼이 **현재 렌더 중인 일반 쿠폰 카드의 실측값**(서브텍스트 1행 + 만료일 1행이 모두 표시된 상태)을 기준으로 동일하게 재현한다. Figma 수치를 별도로 추가하지 않으며, 이 규범은 Figma 원본과 상충하지 않는다(Figma는 일반 쿠폰 카드 형식을 따른다고 명시).
- **점선·하단 영역 여백**: 점선 자체의 상하 마진(`my-[12px]` 유사 패턴) + 점선 아래 컨텐츠 행 높이 + 카드 바닥 패딩을 일반 쿠폰 카드와 동일하게 유지한다. 스킬 이용권 카드에 한해 점선 상하 간격을 단축하지 않는다.
- **구현 주의 — iOS 플랫폼 정합성**: ISS-041 처리 과정에서 iOS `CouponItemCell.bind()`는 스킬 이용권 판정 시 `descriptionLabel.isHidden = true` + `flex.isIncludedInLayout = false`(행 collapse)로 서브텍스트 행을 제거한 상태이다. 본 규범("reserve 방식")과 상충한다. 향후 iOS 파트에서 본 규범에 맞춰 `isIncludedInLayout = true` 유지 + `alpha 0`(또는 동등한 invisible placeholder)로 전환하는 후속 정합 작업이 필요하다. **문서상 규범은 reserve 방식으로 확정**하며, iOS 코드 정합 시점은 별도 과업으로 tracks한다. 레거시 collapse 구현이 잔존하는 기간에도 카드 사용에 기능적 회귀는 없다.

### iOS (hellobot_iOS) — 미착수

- **폰트**: 앱 기본 폰트는 Apple SD Gothic Neo, 팝업 내부는 Pretendard
- **에셋**: Figma에서 @1x/@2x/@3x 규격으로 export 필요
- **팝업 구현**: UIKit 기반 커스텀 모달 또는 기존 팝업 컴포넌트 활용
- **토스트**: 기존 앱 토스트 컴포넌트가 있으면 재사용, 없으면 스타일 맞춰 구현
- **이용권 카드**: 기존 쿠폰 카드 UI에 "이용권" 라벨 + "스킬 보러가기" 링크 추가
- **미로그인 처리**: 로그인 안내 배너 (노란색, X 닫기) + 입력 시도 시 로그인 화면 직접 이동 (팝업 없음)

### Android (hellobot_android) — 개발중

- **폰트**: Pretendard (팝업), 시스템 폰트 (앱 기본)
- **에셋**: Figma에서 mdpi/hdpi/xhdpi/xxhdpi/xxxhdpi 규격으로 export 필요
- **팝업 구현**: DialogFragment 또는 Compose Dialog
- **dp 변환**: 디자인 기준 375px 폭 → dp로 비율 변환 (288px → ~288dp 그대로 사용 가능, 모바일 기준)
- **컬러**: colors.xml에 디자인 토큰 컬러 추가 또는 기존 매핑 확인
- **이용권 카드**: RecyclerView 내 기존 쿠폰 카드 ViewHolder에 이용권 타입 분기 추가
- **미로그인 처리**: 로그인 안내 배너 (노란색, X 닫기) + 입력 시도 시 로그인 화면 직접 이동 (팝업 없음)
