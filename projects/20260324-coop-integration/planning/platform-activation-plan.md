# 카카오 선물하기 플랫폼 활성화 방안 — 이니셔티브 초안

## **Problem**

- 카카오 선물하기에 1차 10종(5/26) + 2차 20종(6/12) 총 30종 라인업 출시를 완료했으나, 출시 후 18일간 실적이 **구매+쿠폰 등록 합산 8건 (내부 구매 포함)** — 일평균 0.5건 미만으로 외부 유입이 사실상 없다.
- 원 이니셔티브(1pager)의 핵심 가설 "거대한 카카오 사용자 풀에 헬로우봇을 노출할 수 있다"가 **노출 단계에서 막혀 검증 자체가 불가능한 상태**. 유입된 사용자가 구매자라는 채널의 강점은 노출이 전제일 때만 성립한다.
- [exposure-strategy §3.4](kakao_product_skills/exposure-strategy.md) D+7 게이트 기준(Go: 일평균 등록 ≥ 20건 / Pivot: < 5건/일)으로 보면 현재는 **명백한 Pivot 구간** — "L4(단독 SKU)·L5(MD 캠페인) 재협상" 트리거가 발동된 상황.
- 수수료 모델(전환 시만 8%)이라 직접 손실은 없지만, 30종 기획·디자인·등록에 투입된 리소스와 운영 유지 비용이 회수되지 않고, 채널 가설 검증이 지연될수록 타 채널 확장(네이버·라인 등) 판단도 함께 밀린다.

_root cause_ — `[검증 필요]` 가설 4건, 실태 진단(Solution 0)으로 확인

1. **부스트 레버(L4·L5) 미실행** — 쿠프마케팅 3/20 회신에서 "노출 배너 활용 카카오 측과 협의 완료, **프로모션 진행 스킴 및 일정 확인 이후 세부 협의 예정**"이라 했으나, 우리가 프로모션 스킴을 제시하지 않아 협의가 멈춰 있는 상태로 추정. 론칭 시점 할인 프로모션(노출 구좌 확보 패키지)도 미진행. `[검증 필요: 쿠프 측 협의 현황·구좌 가용성]`
2. **베이스라인 노출(L2·L3) 미점검** — 검색("운세"·"사주"·"헬로우봇")·카테고리·선물테마탭에서 우리 상품이 실제로 어디에 어떻게 노출되는지 실측한 적 없음. 운세 콘텐츠 카테고리 자체가 카카오에 신생이라 사용자의 자연 탐색 동선이 없다. `[검증 필요: 매대 실측]`
3. **자체 채널 발신(L6) 부재** — 헬로우봇 사용자에게 카카오 선물하기 입점 사실을 알린 적 없음 (카톡채널·앱 배너·SNS 발신 0회 추정). `[검증 필요: CRM 발신 이력]`
4. **신규 브랜드 콜드스타트 악순환** — 카카오 매대 노출 알고리즘은 판매·클릭 실적 기반인데, 초기 실적 0 → 노출 0 → 실적 0 루프. 외부 부스트(프로모션·구좌·자체 트래픽) 없이는 자연 돌파가 구조적으로 어렵다.

## **Customer Job**

1. 회사는 카카오 선물하기가 "지속 가능한 신규 구매자 유입 채널"이라는 가설을 검증할 수 있는 **최소한의 노출량을 확보**하고, 검증 결과에 따라 확장/보수화를 판단하고 싶다.
2. (최종 사용자) 운세 콘텐츠를 선물하고 싶은 카카오 사용자가 선물하기 탐색 동선(검색·테마·기획전)에서 헬로우봇 상품을 **발견할 수 있어야** 한다.

## **Solution / Feature**

활성화 레버 4종 + 선행 진단. 실행 순서: **0 진단 → 1 카카오 부스트 → 2 베이스라인 최적화 → 3 자체 발신 → 4 측정 상시화** (1이 본 방안의 주력, 2·3은 1의 효과를 받치는 베이스).

0. **노출 실태 진단** (선행, 즉시)
   - 카카오 선물하기 앱에서 실측: 브랜드 검색("헬로우봇") / 키워드 검색("운세"·"사주"·"신점"·"타로") 노출 여부·순위, 카테고리·테마탭 내 위치, 상품 상세 진입 동선
   - 쿠프마케팅(이정우)에 요청: ① 카카오 측 노출 현황 (현재 우리 상품이 어떤 구좌에 노출 중인지) ② 상품별 판매 데이터 (구매 시점 기준 — 우리 DB는 쿠폰 등록만 보임) ③ 3/20 협의된 노출 배너의 현재 가용 구좌·조건
1. **카카오 노출 부스트 — 론칭 프로모션 패키지 (L5+L4)** ★ 주력
   - 3/20에 이미 카카오 협의가 완료돼 있던 카드를 실행으로 옮긴다: **신규 상품 할인 프로모션 스킴·일정을 우리가 확정해 쿠프마케팅에 제시** → 노출 구좌(배너) + 기획전 입점 세부 협의 개시
   - 쿠프 제안 예시 스킴: 10% 할인 (예: 24,000원 → 21,600원)
   - `[사용자 결정]` 대상 SKU (전체 30종 vs 대표 5~10종), 할인율 (10% 기준), 기간 (2주 권장), 시점 (6월 말 vs 7월 시즌 기획전 연계)
   - 단독 SKU 협상(L4)은 프로모션 협의 테이블에 함께 올려 노출 가산점 확보 — [exposure-strategy §3.2](kakao_product_skills/exposure-strategy.md) 권장: 결혼 궁합 + 3번의 기회
2. **검색·매대 베이스라인 최적화 (L2+L3)**
   - 30종 전수: 상품명 검색 키워드 점검 ([08-kakao-search-keywords.md](kakao_product_skills/08-kakao-search-keywords.md) 기준), 선물테마 매핑 누락 확인, 섬네일·카피 부진 SKU 보강
   - 비용 0 (등록 정보 수정) — 진단(0)에서 누락 발견 시 즉시 실행
3. **자체 채널 역방향 유입 (L6)**
   - 헬로우봇 앱 배너·카톡채널·SNS로 "카카오 선물하기 입점" 발신 — **"선물" 맥락 한정** (생일·기념일에 친구에게 운세 선물). 본인 결제 유도가 아니라 선물 수요 환기로 잠식 최소화
   - 목적 이중: ① 직접 판매 ② 초기 판매 실적을 만들어 카카오 매대 알고리즘 콜드스타트 돌파
   - `[사용자 결정]` 발신 채널 범위·시점 (프로모션 기간과 동기화 권장)
4. **실적 측정 상시화**
   - 주간 모니터링 고정: 상품별 등록·취소·매출 SQL (1pager Success Metric 첨부 쿼리, `coop_marketing_coupon_usage` 기준) + 쿠프마케팅 판매 데이터 정기 수신 (주간 권장 `[사용자 결정]` 주기)
   - funnel 완성 조건: 분모(카카오 노출·판매)는 쿠프/카카오 데이터, 분자(등록·신규 구매자)는 우리 DB — 양쪽 결합해야 "구매→등록 전환율" 측정 가능 ([data-measurement-plan.md](../data-measurement-plan.md) 정의 승계)
   - 현재 "8건"의 집계 기준 확정: 구매(카카오) vs 등록(우리 DB) 구분, 내부 구매 분리 방법 `[검증 필요]`

## **Success Metric**

input metric

1. **카카오 매대 노출 확보** — 기획전·배너 입점 건수, 브랜드/키워드 검색 노출 순위 (진단 시 베이스라인 기록 → 프로모션 후 비교)
2. **카카오 판매량** (쿠프 데이터, 주간) — 노출→구매 전환의 1차 신호

output metric

1. **쿠폰 등록 건수 (외부 구매분, 주간)** — 현재 베이스라인: 18일 누적 8건(내부 포함). 내부 구매 분리 후 외부분만 추적
2. **앱/웹 신규 구매자수** (1pager 승계 — 등록 경로로 첫 유료 이용 발생 사용자)

판정 기준 (프로모션 종료 후 D+30 게이트, [exposure-strategy §3.4](kakao_product_skills/exposure-strategy.md) 재적용):

- **Go** (Phase 2 확장 검토): 프로모션 기간 일평균 등록 ≥ 20건, 종료 후에도 일평균 ≥ 5건 유지
- **Watch**: 프로모션 중 반응 있으나 종료 후 < 5건/일 → 시즌 기획전 정례화(L5 반복)로 운영
- **Pivot** (채널 보수화): 프로모션 중에도 일평균 < 5건 → Unhappy Path 실행
- `[사용자 결정]` 게이트 수치가 현실적인지 — 8건 베이스라인 대비 일 20건은 높은 목표. 1차 판정선 조정 여지

## **Benchmark**

- 카카오 선물하기 내 콘텐츠 상품군은 영화 티켓 외 부재 — 운세 디지털 콘텐츠는 헬로우봇이 **First Mover** (카카오 측이 카테고리 확장을 위해 먼저 제안한 구도). 노출만 확보되면 경쟁 없는 매대.
- 운세·부적 카테고리 기존 상품은 명태·오행 액막이 등 실물 굿즈 중심 (79.3%, [리포트C](kakao_coupon_product/카카오 선물하기 리서치 기반/리포트C_카카오선물하기_운세부적_상품_분석.md)) — 디지털 운세 콘텐츠 영역은 단독 점유.
- 경쟁/유사 사례 상세: [competitor-research-20260515](kakao_product_skills/competitor-research-20260515/readme.md)

## **Trade off**

- **할인 프로모션 마진**: 수수료 8% + 할인 10% → 실수령 정가의 약 83% (0.90 × 0.92). 단, 노출 비용이 0인 채널에서 유일하게 돈으로 노출을 사는 레버이며 캠페인 기간 한정.
- **자체 채널 발신의 잠식**: 자사 유저가 인앱/웹 결제 대신 카카오 경유 시 8% 수수료 발생. 다만 ① "선물" 수요는 인앱에 없는 신규 수요 ② 인앱 결제는 스토어 수수료(15~30%)가 이미 발생 중이라 카카오 8%가 반드시 손해는 아님 `[검증 필요: 채널별 실효 마진 비교]`. 발신 메시지를 선물 맥락으로 한정해 본인 결제 이전 최소화.
- **운영 리소스**: 프로모션 등록·정산 검증·주간 모니터링 공수 — TODO-029 운영 가이드 체계에 흡수.

## **Unhappy Path**

- 프로모션 + 노출 구좌 확보 후에도 게이트 Pivot (일평균 < 5건) → **채널 가설 보수화**: 신규 SKU·프로모션 투자 중단, 라인업 유지만 (수수료 모델이라 유지 비용 낮음), 쿠프 인터페이스 재사용 가능한 타 채널(네이버 기프트·라인 기프트)로 검증 무대 이동 검토.
- 카카오 측 노출 구좌 협의 불발 (오픈 시기·구좌 사정) → 7~8월 휴가 / 9~11월 수능 시즌 기획전 일정으로 재도전 ([exposure-strategy §4.1.C](kakao_product_skills/exposure-strategy.md) 시즌 캘린더, MD 협상 T-1~2개월 역산).
- 자체 발신이 직접 판매만 만들고 카카오 매대 노출에 영향 없음이 확인되면 → L6는 순수 판매 채널로만 평가하고 노출 돌파는 L5 반복에 의존.

## **Feedback loop**

1. **주간**: 등록·취소·매출 모니터링 (SQL) + 쿠프 판매 데이터 → 주간 리뷰 (이상 신호 시 즉시 액션)
2. **캠페인 단위**: 프로모션 종료 시 회고 — 구좌별 노출→판매 효과 분해 → 다음 캠페인(시즌 기획전) 설계에 반영
3. **게이트**: 프로모션 종료 +30일 Go/Watch/Pivot 판정 → Go면 [exposure-strategy §4](kakao_product_skills/exposure-strategy.md) Phase 2(라인업 확장·단독 SKU·시즌 정례화) 진입, Pivot이면 Unhappy Path
