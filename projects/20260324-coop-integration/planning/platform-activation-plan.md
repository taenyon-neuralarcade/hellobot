# 카카오 선물하기 플랫폼 활성화 방안 — 이니셔티브 초안 (v2)

> 작성일: 2026-06-12 · 작성자: 코디네이터 초안 (사용자 확정 전 draft)
> 접근: 카카오·쿠프마케팅과의 협의가 필요한 일(배너, 기획전, 할인 행사)은 시점과 조건이 외부에 달려 있다. 그래서 이 방안은 **우리 손으로 바로 할 수 있는 일을 먼저 실행**하고, 외부 협의는 내부 실행으로 만든 실적과 데이터를 가지고 이후 소통으로 풀어가는 구조로 짠다.
> 표기: `[확인 필요]` = 추정 상태, 점검으로 확인 후 확정. `[결정 필요]` = 사용자 확정 필요.

## **Problem**

- 카카오 선물하기에 상품 30종(1차 10종 5/26 + 2차 20종 6/12)을 모두 출시했지만, 출시 후 18일 동안 실적이 **구매+쿠폰 등록 합산 8건(내부 구매 포함)** — 하루 평균 0.5건이 안 되고, 외부 사용자 유입은 사실상 없다.
- 이 채널의 원래 기대는 "카카오의 거대한 사용자 풀에 헬로우봇을 노출한다"였다. 지금은 노출 자체가 안 되고 있어서, 채널이 통하는지 검증조차 못 하는 상태다.
- 출시 전 세워둔 성과 판정 기준(하루 등록 20건 이상이면 확장 검토, 5건 미만이면 전략 재검토)에 비춰 보면 명확한 **재검토 구간**이다.
- 판매가 없으면 비용도 없는 구조(판매액의 8% 수수료만 발생)라 직접 손실은 없지만, 30종 기획·디자인·등록에 들어간 리소스가 회수되지 않고 있고, 이 채널의 성패 판단이 늦어질수록 다음 채널 확장 판단도 같이 밀린다.

_root cause_

1. **출시까지만 하고, 알리는 활동을 하지 않았다** — 출시 이후 헬로우봇 앱·푸시·카카오톡 채널·SNS 어디에서도 "카카오 선물하기에서 헬로우봇을 선물할 수 있다"는 안내를 내보낸 적이 없다. `[확인 필요: 발신 이력 0회가 맞는지]`
2. **카카오 안에서 우리 상품이 어디에 보이는지 확인한 적이 없다** — "헬로우봇"이나 "운세"로 검색하면 몇 번째에 나오는지, 어떤 카테고리·선물테마에 들어가 있는지 실측 기록이 없다.
3. **카카오 매대는 잘 팔리는 상품을 더 보여주는 구조다** — 판매·클릭 실적이 없으면 노출이 안 되고, 노출이 안 되니 실적이 안 쌓이는 악순환. 신규 입점 브랜드가 가만히 있으면 저절로 풀리지 않는다.
4. **노출을 직접 끌어올리는 수단(배너·기획전·할인 행사)은 외부 협의 영역이다** — 쿠프마케팅이 3월에 "프로모션 계획과 일정을 주면 카카오와 노출 배너를 세부 협의하겠다"고 안내했으나 아직 진행 전이고, 진행하더라도 시점·자리 확보가 카카오 사정에 달려 있다. 이것만 기다리고 있을 수는 없다.

## **Customer Job**

1. 회사는 외부 협의 없이도 **지금 우리 힘으로 만들 수 있는 노출과 초기 판매 실적**을 만들어서, 카카오 선물하기가 신규 구매자 유입 채널로 작동하는지 판단할 근거를 확보하고 싶다.
2. (사용자 관점) 헬로우봇을 이미 쓰는 사람이 "친구에게 운세를 선물한다"는 새 행동을 알게 되고 쉽게 실행할 수 있어야 하고, 카카오에서 선물을 찾는 사람이 검색으로 우리 상품을 발견할 수 있어야 한다.

## **Solution / Feature**

원칙: **내부 실행 → 실적·데이터 축적 → 그 결과를 들고 외부 협의**. 1~5가 내부 실행 패키지이고, 외부 협의가 필요한 항목은 마지막에 "준비만" 상태로 분리해 둔다.

1. **현재 노출 상태 점검 (기준점 만들기, 즉시)**
   - 카카오 선물하기 앱에서 직접 확인: "헬로우봇" 검색 결과, "운세"·"사주"·"타로"·"신년운세" 같은 일반 검색어에서의 노출 여부와 순위, 카테고리·선물테마 어디에 들어가 있는지
   - 화면 캡처로 기록해 두고, 이후 모든 개선 활동의 전·후 비교 기준점으로 사용
2. **자체 채널로 알리기 (핵심 실행)** ★
   - 헬로우봇 앱 배너·푸시, 카카오톡 채널, SNS에 "카카오 선물하기에서 헬로우봇 스킬을 선물할 수 있어요" 발신
   - 메시지는 "선물 상황" 중심으로: 생일·기념일·시험·이직·연애 고민인 친구에게 — 본인 결제를 카카오로 옮기라는 게 아니라, 선물이라는 새 수요를 여는 방향
   - 노리는 효과 두 가지: ① 직접 판매 ② 초기 판매 실적을 만들어 카카오 매대 노출 순위를 끌어올리는 마중물
3. **앱·웹 안에 "선물하기" 진입점 만들기 (개발 필요 — 반응 보고 단계 투입)**
   - 스킬 상세나 결과 공유 화면에 "이 스킬 선물하기" 진입점을 만들어 카카오 선물하기 상품 페이지로 연결
   - 개발 리소스가 들어가므로 2번 발신의 반응을 보고 투입 여부 결정 `[결정 필요]`
4. **상품 소재 개선안 만들기 (작성은 내부, 적용은 등록 정보 수정 요청)**
   - 상품명에 사람들이 실제로 검색하는 단어가 들어가는지 점검(예: "연애운"·"사주"·"신점"), 섬네일·설명을 "선물 받는 사람" 시점으로 다듬기
   - 개선안 작성은 완전한 내부 작업. 적용은 상품 등록 정보 수정이라 쿠프마케팅에 요청해야 하지만, 협상이 아니라 일상 운영 요청 수준이라 외부 의존이 낮다
5. **실적 측정 루틴 만들기 (상시)**
   - 주 1회 상품별 등록·취소·매출 집계 — 우리 DB의 쿠폰 등록 데이터 기준, 이미 만들어 둔 집계 쿼리 사용
   - 현재 "8건"의 내역 확정: 내부 구매와 외부 구매 구분, 어떤 상품이었는지 → 이후부터는 외부분만 따로 추적
   - 한계 인지: 우리 DB에는 "쿠폰을 등록한 순간"만 남는다. 카카오에서 몇 건 팔렸는지(등록 전 단계)는 쿠프마케팅 데이터를 받아야 보인다 → 외부 협의 항목

**(외부 협의 — 이번 방안에서는 실행하지 않고 준비만)**

- 노출 배너·기획전 입점·신규 상품 할인 행사: 쿠프마케팅이 3월에 "프로모션 계획·일정을 주면 카카오와 세부 협의하겠다"고 한 카드. 내부 실행으로 판매 추이·반응 데이터가 모이면 그걸 근거로 우리 쪽 계획을 제시한다 — 빈손으로 협의하는 것보다 카드가 세진다.
- 판매·노출 데이터 정기 수신: 카카오에서의 판매량·노출 현황을 주기적으로 공유받는 체계 협의.

## **Success Metric**

input metric

1. 자체 발신 도달·클릭 (채널별 발송 수 → 클릭 수)
2. 카카오 선물하기 안에서의 검색 노출 변화 (1번 점검 기준점 대비, 수기 확인)

output metric

1. **주간 쿠폰 등록 건수 (외부 구매분)** — 현재 기준점: 18일 누적 8건(내부 포함)
2. **카카오 경유 신규 구매자 수** — 쿠폰 등록으로 헬로우봇 첫 유료 이용을 시작한 사용자 (앱/웹 구분)

1차 판정 (자체 발신 시작 후 4주):

- 주간 등록이 의미 있게 늘면(예: 주 10건 이상 `[결정 필요: 판정선]`) → 내부 레버 지속 + 외부 협의(배너·기획전)를 실적 데이터 들고 착수
- 발신할 때만 반짝 늘고 유지가 안 되면 → 자체 발신은 직접 판매 수단으로만 평가하고, 노출 돌파는 외부 협의 없이는 어려운 채널로 판단
- 거의 변화가 없으면 → Unhappy Path

## **Benchmark**

- 카카오 선물하기에서 콘텐츠 상품은 영화 티켓 정도가 전부였고, 카카오가 먼저 콘텐츠 상품군을 늘리고 싶어 우리에게 제안한 구도다. 운세 디지털 콘텐츠는 사실상 우리가 첫 입점이라, 노출만 풀리면 경쟁 없는 매대다.
- 카카오 안의 운세 관련 상품은 부적·풍수 소품 같은 실물 굿즈가 약 80%를 차지한다. "운세 콘텐츠를 선물한다"는 행동 자체가 카카오 사용자에게 아직 낯설다는 뜻이고, 그래서 이미 운세 콘텐츠를 소비하고 있는 우리 사용자에게 먼저 알리는 게 출발점으로 합리적이다.
- 내부 리서치 요약: 카카오 선물하기에서 새로 구매를 시작하는 사람의 70% 이상이 "먼저 선물을 받아본" 경험에서 출발하고, 4회 받아본 사람은 30% 이상이 구매자로 전환된다. 초기에 선물 보내는 사람을 만들어내면, 받은 사람이 다음 구매자가 되는 연쇄가 이 플랫폼의 기본 성장 동선이다.

## **Trade off**

- **자기 잠식**: 우리 유저가 인앱·웹 결제 대신 카카오에서 사면 판매액의 8% 수수료가 새로 생긴다. 다만 ① "선물"은 인앱에 없던 수요라 새 매출에 가깝고 ② 인앱 결제는 이미 스토어 수수료 15~30%를 내고 있어서 카카오 8%가 반드시 손해는 아니다 `[확인 필요: 채널별 실제 마진 비교]`. 발신 메시지를 선물 상황으로 한정해 본인 결제 이탈을 줄인다.
- **발신 자원 소모**: 푸시·카카오톡 채널 메시지는 보낼 수 있는 횟수가 한정된 자원 — 기존 마케팅 캠페인 일정과 발신 슬롯 조율 필요.
- **개발 리소스**: 앱 내 선물하기 진입점은 개발 투입이 필요 → 발신 반응을 확인한 뒤 단계적으로 투입해 리스크를 줄인다.

## **Unhappy Path**

- 자체 발신·소재 개선으로도 4주간 등록이 거의 늘지 않으면 → "우리 유저에게 알리는 것만으로는 안 되는, 외부 노출 없이는 못 크는 채널"로 판단. 그때는 ① 외부 협의(배너·기획전·할인)를 본격 추진하거나 ② 채널 우선순위를 낮추고 상품 유지만 한다 (판매가 없으면 비용도 없는 구조라 유지 부담은 낮다).
- 발신이 직접 판매로는 이어지는데 카카오 안 노출 순위는 그대로면 → "초기 실적이 노출을 끌어올린다"는 마중물 가설은 기각하고, 자체 발신은 판매 수단으로만 운영.
- 외부 협의로 넘어가도 카카오 자리 사정으로 일정이 안 잡히면 → 수능(9~11월)·신년(11~1월) 같은 선물 수요가 큰 시즌에 맞춰 재시도.

## **Feedback loop**

1. **주간**: 등록·취소·매출 집계 리뷰 (매주 월) — 이상 신호가 보이면 즉시 대응
2. **발신 단위**: 채널별 발송 → 클릭 → 등록 흐름 회고 → 다음 발신의 메시지·채널 조정
3. **4주 판정**: 내부 레버 지속 / 외부 협의 착수 / 우선순위 하향 결정. 결과가 어느 쪽이든 여기서 모인 데이터가 이후 외부 협의의 협상 카드가 된다
