## ★ 발견 1 — 마트에 first/second/last paid 속성 컬럼군 완비 (132컬럼)

브리프가 가정한 채널 판별 컬럼이 **실제로 존재** (사용자 지적이 정확했음). 본진 마트에 첫/두번째/마지막 구매가 통합·앱별로 denormalize 되어 있음:

| 개념 | 통합(웹+앱) | 앱 전용 (IOS/ANDROID) |
|---|---|---|
| 첫구매 일자 | `user_first_paid_date` (DATE) | `user_first_app_paid_date` (DATE) |
| 첫구매 SKU | `user_first_paid_menu_seq/name/content_type` | `user_first_app_paid_menu_seq/name/content_type` |
| 첫구매 매출 | `user_first_paid_revenue_krw` (FLOAT) | `user_first_app_paid_revenue_krw` (FLOAT) |
| 2회차 일자 | `user_second_paid_date` | `user_second_app_paid_date` |
| 2회차 SKU/매출 | `user_second_paid_menu_*` / `_revenue_krw` | `user_second_app_paid_menu_*` / `_revenue_krw` |
| 마지막구매 | `user_last_paid_*` | `user_last_app_paid_*` |
| 결제일수 | `user_number_of_paid_date` (INTEGER) | — |
| 인구통계 | `user_age` `age_group` `age_group_5yr` `user_gender` | (동일) |

→ 코호트·재구매·SKU transition 분석을 이 단일 마트로 대부분 처리 가능.

## ★ 발견 2 — 통합/앱 첫구매 컬럼의 소스 정의가 다름 (채널 판별에 직접 영향) ⚠

SQL 정의(`union_mart_user_key_actions.sql`) 분석 결과, 두 컬럼은 **모집단·소스가 다름**:

| 컬럼 | 소스 CTE | 포함 결제 | 플랫폼 |
|---|---|---|---|
| `first_paid_date` (통합) | `union_user_actions` MIN(결제일) (L575~581) | `pay_for_*` **+ `in_app_purchase`/`purchase`** | 전 플랫폼 |
| `first_app_paid_date` (앱) | `app_pay_for_skill_with_type` order=1 (L466~474) | `pay_for_*` (스킬결제)**만** | IOS/ANDROID만 |

**→ "통합 ⊇ 앱" 포함관계가 성립하지 않음.** 통합은 인앱구매(하트충전 등)를 포함하고, 앱은 스킬결제만 포함하므로 `first_paid_date > first_app_paid_date` 인 모순 케이스(X_anomaly)가 발생.

이 때문에 **두 컬럼 비교 기반 채널 판별은 근사치**이며, 정밀 분류는 raw 결제이벤트(`mart_use_skill_se` + `mart_purchase_fb`)에서 플랫폼·결제유형별로 재계산해야 함.

### 해석 — ⚠ 주의: C가 최대라는 점이 핵심 반전

당초 가설은 "A 앱퍼스트가 압도적 다수"였으나, **실제로는 C 웹온리(앱 스킬결제 이력 없음)가 59.8%로 최대**. 단 이는 채널 판별 컬럼의 정의 때문일 수 있음:
- C 정의 = "앱 **스킬결제**(`pay_for_*`)가 없음". 하지만 이들도 **앱에서 인앱구매(하트충전)는 했을 수 있음** (통합 first_paid_date 존재).
- 즉 C는 "웹에서만 결제"가 아니라 "**앱 스킬결제 경험이 없는 사용자**"에 가까움. 진짜 웹온리인지 구분하려면 결제 플랫폼 raw 확인 필요.

1. **A 앱퍼스트(35.0%)** — 재구매율 60.5%(any)≈57.1%(app), 채널 일관. 본 분석 메인.
2. **B 웹→앱(3.6%) = 초고 리텐션** (H7 검증) — 재구매율 **92.3%**. 채널 이동이 강한 리텐션 신호. N=44,270 으로 격리 분석 시 적정 규모.
3. **C 웹온리(59.8%)** — 정의 모호성 있음. 후속 트랙으로 분리하되, "앱 스킬결제 미경험"의 정확한 의미를 Phase 1.1 raw 검증으로 확정 필요.
4. **X 이상치(1.6%)** — 컬럼 정의 불일치 산물. 마트 컬럼 비교로는 채널 분류 불가능한 케이스.

### 스코프 결정 → β 채택 (사용자, 2026-05-31)

마트 컬럼 비교 채널판별이 부정확(X_anomaly 1.6%)함이 드러나 **β(raw 정밀 재계산)** 선택. 아래 Phase 1.1(β) 로 재분류 완료.

### α(마트컬럼) vs β(raw) 비교 — 왜 재계산이 옳았나

| 채널 | α 비중 | α 재구매(any) | β 비중 | β 재구매(any) | β 재방문 |
|---|---:|---:|---:|---:|---:|
| A 앱퍼스트 | 35.0% | 60.5% | 37.2% | 79.3% | 62.1% |
| B 웹→앱 | 3.6% | 92.3% | 3.7% | 100%* | 94.6% |
| C 웹온리 | 59.8% | **14.7%** | 59.1% | **23.3%** | 15.1% |

- **C 재구매율 14.7%→23.3% 상향**: α의 C 재구매율은 `second_app_paid_date`(앱 2회차)만 봐서 웹 재구매를 누락한 착시. β는 웹 재구매까지 포착.
- 채널 비중(A 35→37%, B 3.6→3.7%, C 59.8→59.1%)은 양 방식 거의 동일 → 분류 자체는 견고. 차이는 주로 **재구매 측정 방식**.

### 해석

1. **A 앱퍼스트(37.2%) = 본 분석 메인** — 재구매(any) 79.3%, 재방문 62.1%, **동일일만 17.3%**(A·C 중 최고). 즉 A는 재방문 리텐션 + 인세션 업셀 둘 다 활발. 본 분석의 핵심 타겟.
2. **B 웹→앱(3.7%) = 최강 리텐션 (H7 검증)** — 재방문 재구매 **94.6%**. 웹에서 한번 사고 앱으로 넘어온 사람은 거의 다 앱에서 또 산다. N=45,526 격리 분석 적정. (동일일만 5.4%로 낮음 → B의 재구매는 재방문 중심)
3. **C 웹온리(59.1%) = 후속 마케팅 트랙** — 콘텐츠 구매자 과반이 앱 결제 이력 없는 웹 전용. 재구매율 23.3%로 A의 1/3. 본 분석(인앱 추천) 레버 밖.

### 스코프 최종 확정 (β 기준)

- **A 앱퍼스트(458,291)** = 메인 풀분석
- **B 웹→앱(45,526)** = 격리 분석 (N 제약 내 세분화)
- **C 웹온리(727,159)** = 후속 마케팅 트랙 (규모만 기록)
- U 미상(2) = 제외

### ★ 반복성 = 콘텐츠 특성 vs 가격 특성 (사용자 질문 — 교란 분리)

방법: **층화(stratification)**. 가격tier(T1≤4k/T2 4-8k/T3 8-12k/T4>12k) × content_type(타로/사주) 격자에서, *행 비교=가격효과 / 열 비교=콘텐츠효과*. 가격분포: 타로 median 1,500원 / 사주 median 18,000원 (가격이 콘텐츠와 강하게 엉킴 → 층화 필수).

**rev90 격자 (전체 topic, `phase2b_price_x_content.png`)**
| 가격tier | 타로 | 사주 |
|---|---|---|
| T1 ≤4k | 44.2 (n18,894) | 32.0 (n3,205) |
| T2 4-8k | 45.7 (n1,573) | 33.7 (n483) |
| T3 8-12k | 50.4 (n3,992) | 37.8 (n2,817) |
| T4 >12k | 49.3 (n4,896) | 41.5 (n51,423) |

**결론 — 콘텐츠 특성이 주(主), 가격도 부(副)로 기여 (둘 다 실재)**:
1. **콘텐츠 효과 (열 비교, 가격 고정)**: *같은 가격대*에서 타로가 사주보다 항상 +8~12%p 높음 (T1: 44.2 vs 32.0 / T4: 49.3 vs 41.5). → **싸서가 아니라 타로라는 콘텐츠 자체가 반복형.** 가장 비싼 타로(T4 49.3)가 가장 싼 사주(T1 32.0)보다 훨씬 높음 = 가격으로 설명 불가.
2. **가격 효과 (행 비교, 콘텐츠 고정)**: 사주 내에서 T1 32.0 → T4 41.5로 *비쌀수록 오히려 재구매↑* (싼 사주가 덜 반복). 타로도 T1 44 → T3 50으로 비싼 쪽이 높음. → **"싸서 반복"은 틀림.** 오히려 고가 구매자가 더 충성(셀렉션: 비싼 걸 산 사람이 더 진성).
3. **[연애 only] 교란 3중 통제** (`phase2b_price_x_content_love.png`): topic까지 고정해도 패턴 동일 — 타로>사주 유지, 가격↑일수록 rev90↑ (연애-타로 T4 52.8 최고). → 관심사 제거 후에도 콘텐츠 효과 견고.

> 즉 "타로가 반복적인 건 저렴해서"라는 가설은 **기각**. 콘텐츠 형식(타로=반복 소비 구조)이 본질이고, 가격은 *낮을수록 반복*이 아니라 *높을수록 진성 셀렉션* 방향으로 별개 작용.

### Phase 2 종합 (추천 함의)
- **cross-sell 1순위**: 대량 진입하는 사주 구매자(58K)에게 **타로** 콘텐츠 추천 → 콘텐츠 효과로 재구매율 상승 기대. 가격을 낮춰 유도하는 것은 역효과 가능(저가가 반복을 만들지 않음).
- **연령 맞춤**: 20대=썸/새만남/재회(연애), 30대초=결혼/궁합, 40대+=재물/총운. 단 재물·총운은 1회성이라 추천보다 신규 진입 유도 성격.
- **충성 세그먼트**: 고연령 연애 + 고가 타로 + 썸/짝사랑/재회 intent = 재구매 최상위 결합.

2. **속마음 ↔ 썸짝 양방향 클러스터** (속마음→썸짝 3.44, 썸짝→속마음 2.46) — 초기 연애(짝사랑·상대 속마음 탐색)가 서로를 부름. **초기연애 사용자에게 상대 속마음/썸 콘텐츠 교차 추천이 데이터로 지지**.
3. **재회는 고립** (재회→재회 2.64, 그 외 전부 <0.6) — 재회 관심자는 오직 재회만 반복. 추천도 재회 후속물이 정답.
4. 연애일반·새만남은 서로 호환(1.3대) — 솔로/신규만남 계열은 묶어서 추천 가능.
