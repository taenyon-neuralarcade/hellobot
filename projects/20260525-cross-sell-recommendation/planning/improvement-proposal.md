## 핵심 개선과제 (Top 5)

### 과제 1 — 출력 정제 게이트: "팔 수 없는 것·관심 아닌 것" 제거
- **문제**: rank-1 추천의 **26%가 판매중지·비공개**(을사년 시즌 14건 전부 stop-selling), **24%가 할인·충전·추가구매 SKU**(예: 솔로탈출 1순위=🔋질문권 충전), **9.4%가 자기추천**(방금 산 걸 또). 그대로 노출 시 "구매 불가 링크"·"관심추천에 충전상품" → UX·측정 동시 붕괴(회수불가 푸시는 더 치명적).
- **개선 방향**: 산출 마지막 + 서빙 시점 **하드게이트 3종** — ① 판매상태 `is_open & !is_stop_selling & buyers_30d>0` ② 비콘텐츠/업셀 blocklist(질문권·충전·`(추천할인)`·`[추가구매]`·패키지) ③ 보유·동일스킬 제외. 빈자리는 fallback.
- **기대 효과**: rank-1의 **~35% 즉시 정화**. 슬롯 전환이 "진짜 추천 효과"로 측정됨. 후속 모든 개선의 전제.
- **PO 판단**: **난이도 低·임팩트 高 → 6/9 전 즉시.** 단일 최우선.

### 과제 2 — 신호 정의 교정: affinity를 '관심'으로 되돌리기
- **문제**: 학습 풀의 **68%가 당일(same-day) 구매** = 인세션 번들/업셀. affinity가 "다음 관심"이 아니라 "과거에 같이 노출·끼워판 것"을 학습 → ① 결제직후 슬롯이 과거 배치를 **자기강화(closed loop)** ② 재방문 푸시엔 인세션 업셀물 미스매치(당일 vs 재방문 top5가 **48%만 일치**).
- **개선 방향**:
  - **(단기)** 표면별 분리 산출 — 결제직후=`s_menu`(당일), 재방문 푸시·홈탭=`r_menu`(진짜재방문). "단일 엔진" 폐기.
  - **(근본)** **노출(impression) 로그 기반 lift** — "노출됐는데 안 산 것" 대비로 재정의해 노출 편향과 관심을 분리.
- **기대 효과**: 단기=재방문 추천 정합 + 자기강화 차단. 근본=**개인화 신뢰의 천장 상승**(진짜 관심 신호).
- **PO 판단**: **임팩트 最高.** 단기(표면분리)는 2주 내, 근본(노출로그)은 `/dev-data` 별도 트랙으로 분리 착수.

### 과제 3 — 모집단·채널 정합: 웹엔 웹 데이터
- **문제**: 추천 대상은 '웹 X 구매자'인데 학습은 'A 앱퍼스트'. 연령 `<25` **A 38.7% vs C 18.2%**(2배), 진입스킬 지지율 역전(2141 앱 6배·49816 웹 4배). 근데 **C(웹)가 앵커당 데이터 3~7배**이면서 A와 동일 연애 클러스터.
- **개선 방향**: 채널별(A/C) 분리 산출, **웹 표면엔 C 사용**, 앱↔웹 top-N 교집합만 고신뢰. ("앱=깨끗" 전제는 마케팅유입/오가닉 축을 분리 검증하기 전엔 보류.)
- **기대 효과**: 웹 추천 정확도↑(데이터 3~7배 활용), 모집단 불일치 베팅 제거.
- **PO 판단**: **난이도 低(`c_users.csv` 이미 존재)·임팩트 中~高 → 1주.**

### 과제 4 — 통계 신뢰성: 방향·표본·lift 정정
- **문제**: ① **양방향 pooling이 추천 시간방향 오염** — 정방향-only vs pooled top-1이 **7개 중 5개 다름**(역방향 최대 79%) ② **small-n 무의미** — 웹 3종 nX 4~8(n=1 추천), 12개 중 **10개 top-1이 rank-2와 Wilson CI 겹침**(통계적으로 1위 아님) ③ **lift 분모 오류** — endpoint 빈도 ≠ P(Y as 2nd), 인기 파트너 **1.3~1.5배 오스케일** + self-pair 이중카운트(conf −7%).
- **개선 방향**: 정방향만(`pairs[f==X].s`) + 표본게이트(`pair_n≥30 & anchor_n2≥50` + Wilson 하한, **CSV 산출단계 강제**) + lift 분모=`P(Y as 2nd)` 또는 PMI/Jaccard + conf/lift **단일 기준**.
- **기대 효과**: 추천이 통계적으로 방어 가능, 역방향·노이즈 추천 제거, lift 절대값 신뢰 회복.
- **PO 판단**: **난이도 低(스크립트)·임팩트 中~高 → 1주(과제3과 함께 P0 정제본).**

### 과제 5 — 의사결정 가드레일: 증분은 A/B로만
- **문제**: 적중률 36.6%는 "이미 살 사람"을 맞힌 **상관 수치**(holdout/대조군 컬럼 0). 이를 ROI로 읽으면 노출의 *신규구매 증분*을 과대평가. 게다가 affinity는 2번째 도달자(73.5%)로 학습 — 정작 공략 대상인 1회성 이탈자(26.5%) 취향은 표본에 없음.
- **개선 방향**: 출시=**노출군 vs 비노출군 A/B**, 북극성=첫구매자→2번째 전환율, 적중률은 모델 비교 보조로 강등. 1회성 이탈자엔 인기 fallback 우선 분기.
- **기대 효과**: 과대투자·잘못된 확신 방지, 진짜 증분 확정.
- **PO 판단**: **임팩트 高(전략적)** — 데이터 수정이 아니라 *출시 프로세스 게이트*. P2 인앱 슬롯 출시의 필수 조건.

## ⚠ 과잉투자 경계 (Credit — 하지 말 것)

리뷰가 *반증*한 것 — 여기에 자원 쓰지 말 것:
- **demographic(성별·나이) 개인화** — 연령 age-표준화 lift 변화 **5.1%** = 교란 아님. 세분화 개인화는 ROI 거의 0 (action-priorities §0-4와 동일 결론).
- **복잡한 추천 모델** — co-purchase 단순이 적중률 최강(36.6 vs 정교한 방법들). **모델 고도화보다 정제·가드레일(과제1·4)이 ROI 훨씬 높음.** affinity 순위도 인기로 환원 안 됨(Spearman 0.32) → 방법 자체는 유효, 손볼 건 데이터 위생.
