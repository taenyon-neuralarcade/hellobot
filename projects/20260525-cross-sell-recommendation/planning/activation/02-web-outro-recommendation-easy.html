웹 아웃트로 추천 — 쉽게 보기

웹에서 처음 운세를 산 분에게 결과를 본 직후 "이것도 보세요"를 띄우되, ⓐ 웹에서 바로 하나 더 사게 하거나 ⓑ "앱에서 쓰는 할인쿠폰"을 줘서 앱으로 데려오는 기획이에요.
🔍 자세한 기획서 보기 →
📌 한눈에
  • 누구에게 — 웹에서 처음 결제한 새 손님
  • 어디서 — 결과 확인/결제가 끝난 직후 화면(아웃트로)
  • 두 갈래 — ⓐ 웹에서 바로 추가 구매 · ⓑ 앱 전용 할인쿠폰으로 앱 유도

🎯왜 하나요?

  • 웹 손님은 광고 보고 들어와 운세 하나 딱 사고 떠나는 경우가 많아요 (10명 중 7~8명이 한 번뿐).
  • 이런 분들은 웹에서 계속 붙잡기보다 앱으로 옮겨오면 다시 찾을 가능성이 훨씬 큽니다.
  • 그래서 ⓑ 앱 쿠폰이 핵심 한 수예요.

💡어떤 운세를 추천하나요?

  • 웹에서 ‘함께 산 기록’은 광고·끼워팔기 영향을 받아서 진짜 관심을 잘 못 보여줘요.
  • 그래서 같은 운세를 산 앱 사용자들이 관심 가진 운세를 가져와 추천합니다 (웹 인기 12개는 이미 준비됨).
  • 더 많은 운세로 넓히려면 웹 손님 데이터로 목록을 더 만드는 작업이 필요해요.

🔀ⓐ 바로 구매 vs ⓑ 앱 쿠폰

  • ⓐ 웹에서 바로 추가 구매 → 즉시 매출
  • ⓑ 앱 쿠폰으로 유도 → 앱으로 이사 + 단골화 (길게 보면 더 큼)
  • 권장: 먼저 ⓐ를 권하고, 안 사면 ⓑ 쿠폰을 주는 식 — 또는 둘 다 시험해 보기

🛡️꼭 지킬 안전장치

  • 안 파는 / 품절 운세 추천 금지, 충전권 같은 건 빼기

    ⚠ 쿠폰은 공짜가 아니에요 → 할인해 준 만큼 빼고도 이득인지 꼭 따져야 합니다. 손해면 멈춰요.

📏효과는 이렇게 확인해요

  • 쿠폰 준 그룹 vs 안 준 그룹을 비교해서, 할인 비용을 빼고도 더 벌었는지 확인합니다.
  • 추천을 잘 맞히는 것과 진짜 더 버는 것은 다르니, 비교 실험으로만 판단해요.
💡 쉽게 말하면
웹은 “길거리 1회성 손님”이 많은 가게예요. 계산대에서 ⓐ “이것도 하나 더 어떠세요?” 하고 권하거나, ⓑ “우리 단골앱 깔면 이 쿠폰 드릴게요” 하며 단골로 만드는 것과 같아요.
✅ 결정해 주세요
  • 쿠폰 할인 얼마나? — 손익분기 기준을 먼저 정하기
  • ⓐ와 ⓑ 중 뭘 먼저? — 권장: ⓐ 먼저, 안 사면 ⓑ
  • 앱 안 깐 사람은 어떻게? — 설치를 권할지, 그냥 웹에서 사게 둘지
