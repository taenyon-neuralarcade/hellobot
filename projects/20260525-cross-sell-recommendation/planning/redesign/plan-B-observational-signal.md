## 1. 문제 재정의 — "의미있는 결과"를 관측신호 관점으로

추천 목표는 **첫 구매자의 2번째 구매(1→2 전이) 유도**이고, 학습 신호는 "A를 산 사람의 *진짜 다음 관심* B"다. 구매 co-purchase(P0)는 이미 강한 신호지만 두 교란을 품는다: (a) **인기** — 아무에게나 잘 팔리는 B가 모든 앵커에서 위로 옴, (b) **인세션 노출 결합** — A 직후 같은 세션 화면에 B가 깔려서 산 것(자기강화 closed-loop). 노출(조회) 데이터를 **분모로** 넣는 이유는 단 하나: *"B가 눈앞에 왔는데도(=조회) A를 산 사람이 평균보다 더 많이 B로 전환하는가"*를 물어 인기와 노출 가용성을 상쇄하고 **순수 관심**만 남기려는 것.

### 1-1. POC가 측정한 것 vs 측정하려던 것

| | 측정하려던 것(관심) | POC가 실제 측정한 것 | 결과 |
|---|---|---|---|
| 분자 | A-buyer가 *다른 날* B로 전환 | A-buyer가 조회 후 14일내 B 구매 — **90~95%가 당일(인세션)** | 없애려던 인세션을 재측정 |
| 분모 | A-buyer와 *동급 의향*의 baseline 전환 | *전체* 조회자의 전환(의향 낮은 모집단) | +73% intent floor → null≠1.0 |
| 정규화 | 인기 상쇄 | base_cvr(=인기)로 나눔 | 인기 페널티가 신호를 잡아먹음(partial −0.002) |
| 앵커 | 진짜 첫구매자 | 12m rn=1 (창 이전 결제 보유 40~48%) | 1→2 전이 목표 불일치 |

### 1-2. "의미있는 결과"의 조작적 정의 (관측신호)

본 트랙이 "성공(빌드 GO)"이라고 부를 수 있는 결과는 **단 하나의 양성**이다:

> **인기·인세션·intent-floor·좌측절단을 모두 통제한 뒤에도**, 조회기반 신호(`obs_lift` 또는 PMI)가 구매전용 신호(P0 conf) 대비 **순위를 유의하게 재배열**하고(앵커-내 partial Spearman의 절대값이 0과 분리, §4-2), 그 재배열이 **plausibility 게이트**(교차도메인 < 동일도메인, null=1.0 복원, lift<1 비중 ~50%)를 통과한다.

이 중 하나라도 깨지면 신호는 "관심"이 아니라 잔존 artifact다 → §5 효과0 판정으로 간다.

### 2-3. 세 개의 창 분리 (결함#3 제거의 토대)

POC는 "조회 후 14일내 구매" 한 창에 노출과 전환을 합쳤다. 본 설계는 **노출 관측 창**과 **전환 창**을 분리하고, 전환에서 **당일/동일세션을 잘라낸다**:

```
 f_ts(A 첫구매) ──▶  [lookback 노출창 L]  B 상세조회 관측(고려 신호)
                          │
                          └─▶ [전환창 C, 단 당일 제외]  B 구매 = 재방문 전환만
```

- **노출 관측**: `open_skill_description`(§3-3 확정, A 이후, B≠A). 유저별 B 최초 조회 1건.
- **전환(분자)**: 그 조회 이후 B 구매 중 **`buy_date > view_date`(다른 날)인 것만**. 동일일 전환은 인세션 오염으로 간주해 *분자에서 제외*. 이로써 분자가 master의 `r_menu`(다른날 첫 재방문 결제) 철학과 정합 — POC 90~95% 당일 문제가 구조적으로 사라진다.
  - 보강(세션 정밀도): funnel 테이블에 세션키가 있으면 "동일 세션 OR 동일일" 제외로 강화(§6 피저빌리티에서 컬럼 확인). 세션키 부재 시 **동일일 제외로 충분** — readme §2-2가 당일=68% 인세션을 이미 입증했으므로 "다른 날"이 인세션 차단의 1차 방어선.
- **전환창 C**: 조회 후 다른 날 ~ +30일(여러 길이 7/14/30 민감도). 상한은 master 재방문 정의(90일)보다 짧게 둬 "조회→전환" 인과 근접 유지.

> **이중 대안(권장)**: 노출 게이트 없이 master의 `r_menu`를 **결과 라벨**로 직접 쓰는 단순 버전도 병행한다(§3-D). "노출이 더하는가"의 비교를 위해서는 노출 게이트 버전이 필요하지만, r_menu 버전은 BQ 0회로 즉시 산출 가능하고 인세션·free·앵커 결함이 애초에 없어 **가장 방어가능한 하한**을 준다.

## 3. 신호/지표 정의 (수식) — 6대 결함 1:1 매핑

표기: 앵커 A, 후보 B. `N_A` = 진짜 첫구매 A 앵커 인원(master). `view(B|A)` = A 구매 후 lookback 내 B 상세조회한 A-앵커 유저. `conv(B|A)` = 그 중 다른날 전환창 내 B 구매.

### 3-A. 노출정규화 전환 — intent-floor·인기·인세션 제거형 (`obs_lift`)

POC의 `cvr_from_A/base_cvr`를 버리고 **분모를 "동일 의향 코호트 + leave-out"** 으로 교체한다. 핵심: 분모를 "전체 조회자"가 아니라 **"B를 조회한 *다른* 앵커의 첫구매자들"** 로 잡고, 분자 유저를 분모에서 뺀다.

```
cvr(B|A)      = |conv(B|A)| / |view(B|A)|                      ← A-buyer가 B 조회 후 다른날 전환율
cvr_base(B|¬A)= |conv(B | first-buyer, anchor≠A)| /            ← B를 조회한 "다른 앵커" 첫구매자의 전환율
                |view(B | first-buyer, anchor≠A)|                (분자 유저 leave-out)
obs_lift(B|A) = cvr(B|A) / cvr_base(B|¬A)
```

- **결함#2(intent floor) 제거**: 분모가 더 이상 "전체 조회자"(의향 낮음)가 아니라 **같은 '첫구매자×B조회자'** 모집단이다. 분자(A-앵커)와 분모(¬A-앵커)가 *동급 의향*이므로 +73% floor가 사라지고 **null=1.0이 복원**된다(A가 B와 무관하면 A-앵커 전환율 = 타-앵커 전환율 → lift=1). leave-out으로 자기포함 편향 제거.
- **결함#1(인기) 제거**: 인기 높은 B는 분자·분모 *둘 다* 높아 상쇄된다. POC는 base_cvr(=B의 절대 인기)로 나눠 인기를 *페널티*로 잘못 도입했지만(partial −0.002), 여기선 분모도 "B 조회자의 전환"이라 인기가 약분된다 → 인기 통제 후에도 신호가 0이 아닌지를 §4-2 partial로 직접 검정.
- **결함#3(인세션) 제거**: 분자·분모 모두 "다른 날 전환"만 — 양쪽에서 당일 인세션을 동일하게 제거.

### 3-B. 인기 불변 연관척도 — lift의 대안/병행 (`PMI`, `npmi`, `Jaccard`) (결함#1 정면)

lift는 비율이라 분모 변동에 민감하고 소표본에서 폭주한다(POC base_cvr=0.01에서 lift 10.65). 인기에 강건한 정보이론 척도를 **병행 산출**해 신호가 척도 선택에 불변인지 본다. 모두 **구매 이벤트** 기준(조회 게이트와 별개로, P0와 동일기반 비교용):

```
P(A,B as 1st→2nd) , P(A)=앵커 A 비중 , P(B)=B의 2nd-prior(=P0 base)
PMI(A→B)  = log2( P(B|A) / P(B) )            ← lift의 로그(부호=양/음연관)
npmi(A→B) = PMI / ( -log2 P(A,B) )           ∈ [-1,1] 정규화, 소표본·희귀쌍 강건
Jaccard(A,B) = |buy A ∧ buy B| / |buy A ∨ buy B|  ← 순서무관 동시구매(인기 비대칭 완화)
cond_lift  = P(B|A) / P(B|¬A)                ← 분모가 "A 안 산 사람의 B율"(전체 prior 아님)
```

- **`cond_lift`가 결함#1의 핵심 진단**: P0의 lift는 `P(B as 2nd|A)/P(B as 2nd)`로 분모가 전체 prior(인기)다. `cond_lift`는 분모를 "**A를 첫구매 안 한** 첫구매자의 B-2nd율"로 바꿔 인기를 *반사실(counterfactual)* 로 통제한다. 둘이 크게 갈리는 앵커 = P0 lift가 인기에 오염된 앵커.
- **npmi**가 소표본에서 lift보다 안정적이므로, §4 랭킹 비교의 1차 척도로 lift 대신 **npmi 또는 Wilson-하한 conf**를 쓴다(POC의 lift 폭주 재발 차단).

### 3-C. 표본·다중비교 게이트 (결함#5의 통계 부분)

P0와 동일하게: **앵커 `n ≥ 50`, pair `n ≥ 10`**(조회 버전은 `view(B|A) ≥ 30`). 정렬·순위 결정은 **Wilson 하한**(`build_nbo_p0.py`의 `wilson_lb`)으로, 점추정 lift가 아니라 보수적 하한으로. 다중비교: 앵커당 후보가 수십~수백이므로 "재배열 유의"를 앵커별로 주장할 때 **Benjamini-Hochberg FDR(q=0.10)** 를 적용(아래 §4-2 부트스트랩 p값에). scipy 없음 → BH는 정렬+임계 한 줄로 numpy 구현.

### 3-D. r_menu 직접 라벨 — BQ 0회 하한 (결함 전부를 회피한 보수 기준선)

```
conf_rev(B|A) = |{A-first-buyer : r_menu = B}| / N_A      ← master r_menu (다른날 첫 재방문 결제)
lift_rev(B|A) = conf_rev(B|A) / P(B as r_menu)            ← 재방문 prior로 정규화
```
master CSV만으로 산출(조회 데이터 불요). 인세션(다른날 정의상 제외)·free(VALID 필터)·앵커(master=진짜 첫구매)·좌측절단이 **전부 구조적으로 없다**. 이것이 "노출 트랙이 *이것보다* 더하는가"의 비교 바닥선.

### 3-E. 결함#6(불공정 비교)을 막는 산출 규약

**모든 신호를 동일 코호트(master 98,392)·동일 후보집합(VALID 783)·동일 윈도우에서** 산출한다. 비교 4종을 한 번에 만든다:
| 신호 | 모집단 | 창 | 인세션 | BQ |
|---|---|---|---|---|
| P0 conf/Wilson (기준) | master A | ever-2nd (s_menu) | 포함 | 0회 |
| P0-rev conf (3-D) | master A | r_menu(다른날) | 제외 | 0회 |
| obs_lift (3-A) | master A | 조회후 다른날 전환 | 제외 | 1회(게이트) |
| npmi/cond_lift (3-B) | master A | 동일창 짝 | 해당창 따름 | 0~1회 |

→ 어떤 비교든 "**같은 앵커·같은 창·같은 후보**"에서만 순위를 견준다. POC의 "2년 s_menu vs 12m 조회후구매" 같은 모집단 교차 비교를 금지.

### 3-F. 6대 결함 → 제거 장치 매핑(요약)

| # | 결함 | 본 설계의 제거 장치 | 검증으로 확인(§4) |
|---|---|---|---|
| 1 | 인기 교란(base_cvr로 나눔) | 분모를 "B 조회자 전환"으로(약분) + `cond_lift`(반사실 prior) + npmi | partial Spearman(obs, P0\|인기) ≠ 0 |
| 2 | intent floor(분모 모집단 낮은 의향) | 분모 = 동급 의향(¬A 첫구매·B조회자) + leave-out | **null=1.0 복원** 확인 |
| 3 | 인세션 오염(90% 당일) | 분자·분모 **다른날 전환만**(+세션키 시 동일세션 제외) | 당일 비중 0% 확인 |
| 4 | 좌측절단(앵커 40% 가짜) | master 진짜 첫구매 집합 고정(2021 좌측가드) | 앵커 창-이전 결제 0% |
| 5 | SQL버그·free/WEB | `COUNT(DISTINCT user_id)` + VALID(유료·판매중) + 표본게이트 | viewers 중복 0, free 0 |
| 6 | 불공정 비교 | 동일 코호트·창·후보 4종 동시 산출(§3-E) | 동일기반 churn(§4-1) |

## 4. 검증 설계

POC를 무너뜨린 비판검증을 **설계 단계에 내장**한다. 네 검정 모두 통과해야 "신호 잔존"으로 본다.

### 4-1. 동일기반 랭킹 churn (결함#6 직접 반증)
같은 코호트·창에서 P0 conf 순위 vs obs_lift(또는 npmi) 순위를 앵커별로 Spearman. **POC의 함정**(−0.23이 모집단 차이의 합성물)을 피하려고, 비교 상대를 반드시 **3-D/3-E의 동일기반 P0**로 둔다. 기대: 동일기반이면 POC 재확인대로 −0.06~+0.19 근방이 나올 수 있음 — 그렇다면 그 자체가 "재배열 없음"의 증거.

### 4-2. Partial correlation — 인기 통제 후 잔존 (결함#1 직접 반증)
POC 헤드라인을 죽인 그 검정을 **승격된 신호에 그대로** 적용:
```
partial_Spearman( P0_conf , obs_lift | popularity )      , popularity = B의 2nd-prior(=base)
```
- **0과 분리(부트스트랩 CI 미포함 of 0)** 여야 신호가 인기 부작용이 아님. POC는 여기서 −0.002로 죽었다.
- 앵커-내(within-anchor)와 풀(pooled) 둘 다. 앵커-내가 핵심(앵커 고정 = 의향·시즌 통제). POC 재계산은 앵커-내 +0.17(구매와 관심 *같은* 방향)이었음 — 이게 재현되면 "조회는 구매가 이미 말한 것 외에 추가 정보가 없다"는 강한 효과0 신호.
- 유의성: 앵커별 부트스트랩(B=1,000, user 리샘플) → partial 분포의 CI. scipy 없이 numpy 순열/부트스트랩.

### 4-3. Plausibility 게이트 (결함#2·#3 직접 반증)
신호가 살아있다면 **반드시** 만족해야 하는 정합 조건들(POC는 전부 위반):
1. **null=1.0**: A와 무관한 random B에서 obs_lift 중앙값 ≈ 1.0 (POC ~1.74). 무관쌍 플라시보로 확인.
2. **lift<1 비중 ~50%**: 공정하면 절반은 음의 연관(POC 4.8%만). 비대칭이면 floor 잔존.
3. **교차도메인 < 동일도메인**: 사주→타로 같은 교차가 동일토픽 연속보다 lift 낮아야 자연스러움(POC는 역전 — floor 아티팩트의 지문).
4. **당일 비중 0%**: 분자에서 인세션 제거가 실제로 됐는지 카운트 확인(POC 90~95%).
5. **자기강화 재현 검증**: P0 상위 near-dup pair(예 "속마음→그사람의속마음")가 obs에서 강등되는지 — **단, viewers를 DISTINCT로 고친 값으로**(POC 버그값 0.70은 2.27로 역전했음). 강등이 *버그 아닌 진짜*면 노출 트랙의 유일한 명백한 가치.

### 4-4. 커버리지·취약성
- obs_lift 산출 가능 pair = `view(B|A) ≥ 30` 만족 수. POC는 P0 pair의 20%만 커버(고볼륨 tail) — **측정 universe가 P0의 일부면 "전 앵커 재배열" 주장 자체가 불가**. 커버리지 < 50%면 "노출 트랙은 head 앵커 보강용"으로 범위 축소.
- pair별 조회-커버리지(<50%, 마케팅 직구매 사각) 플래그 — §3-3 미커버 32% 통제.

## 5. 효과 0 시나리오 — "P0로 충분" 판정기준

아래 **판정표**로 빌드 GO / 보류를 기계적으로 결정한다(단일 분석 흥분 금지, §3-7 교훈).

| 검정 | GO(신호 잔존) 조건 | STOP(효과0=P0로 충분) 조건 |
|---|---|---|
| §4-2 partial(인기 통제) | 앵커-내 partial의 부트스트랩 CI가 **0을 포함하지 않고 \|ρ\|≥0.15** | CI가 0 포함 **또는** \|ρ\|<0.10 (POC=−0.002) |
| §4-3 null=1.0 | 무관쌍 중앙값 ∈ [0.9,1.1] | floor 잔존(중앙값>1.2) → 신호 아직 오염 |
| §4-3 lift<1 비중 | ~30–60% | <15% (intent floor 미제거) |
| §4-3 교차<동일 | 성립 | 역전(POC 지문) |
| §4-1 동일기반 churn | 앵커 다수에서 의미있는 재배열(Spearman 중앙 <0.5 **이며** §4-2 동반) | −0.06~+0.19(POC 재확인) = 재배열 없음 |
| §4-4 커버리지 | ≥50% pair 산출 가능 | <30% → 전면 적용 불가, head-only |

**판정 규칙**:
- **GO(노출 트랙 빌드)**: §4-2 partial GO **AND** §4-3 네 게이트 모두 통과 **AND** §4-4 커버리지 ≥50%. 즉 "인기·인세션·floor·좌측절단 다 빼도 조회가 *추가* 정보를 주고, 그게 plausible하고, 충분히 넓다."
- **STOP → P0로 충분**: §4-2가 STOP(가장 흔한 결과로 예상 — POC가 이미 −0.002)이거나, plausibility 한 개라도 깨지거나, 커버리지 <30%. 이때 결론:
  > **노출정규화는 인기 페널티/소표본 잡음만 추가할 뿐, 구매 co-purchase가 담은 신호를 넘지 못한다. 추천은 P0(VALID 게이트·정방향·Wilson 하한 정렬, `build_nbo_p0.py`)로 충분하다.** 노출 트랙은 투자 보류(과잉투자 경계). 노출 데이터는 추천 신호가 아니라 **노출량 진단**(어떤 pair가 노출 부족인지 = A/B 실험 우선순위)에만 활용.
- **부분 GO(head-only)**: partial GO + plausibility 통과지만 커버리지 30–50% → 노출 트랙을 **고볼륨 head 앵커 한정** 보강으로만 빌드, tail은 P0.

> 이 판정은 "노출 *소스*가 유효"(§3-3, 변함없음)와 독립이다. 소스는 좋아도 *신호로서 P0를 못 넘으면* 빌드 안 한다 — 이게 본 계획의 핵심 절제.

## 6. 피저빌리티 · 비용 · 한계

### 6-1. 피저빌리티 게이트 (≤2GB, 설계 확정 전 가벼운 확인만)
설계가 목표이므로 풀 산출 금지. /dev-data 핸드오프 **전** 두 가지만 가볍게 확인(각 dry_run + 파티션필터, 합 ≤2GB):
1. **세션키 존재 확인**: `mart_v2_skill_funnel_fb` 스키마에 session/ga_session_id류 컬럼 유무(동일세션 제외를 동일일 제외보다 정밀화할 수 있는지). 부재 시 §2-3대로 동일일 제외로 진행 — 설계 변경 없음.
2. **obs_lift 커버리지 사전 추정**: 단일 월(예 2026-02) 파티션 1개로 `view(B|A) ≥ 30` 만족 pair 수를 대략 추정(`--project_id=hellobot-f445c`, `event_date` 단일 파티션, `platform IN ('IOS','ANDROID')`). §4-4 커버리지가 애초에 <30%면 **풀빌드 자체가 무의미** → 즉시 STOP 판정.
- **0회 가능 신호 우선**: 3-B(npmi/cond_lift, 구매기반)·3-D(r_menu)는 master/카탈로그 로컬 CSV만으로 BQ 0회. **먼저 이걸로 §4-1·4-2(동일기반 churn·partial)를 돌려** 효과0이 로컬에서 이미 보이면 BQ 조회 빌드를 **아예 안 함**(POC 5.57GB 낭비 반복 차단).

### 6-2. 비용
- 로컬(0 BQ): 3-B·3-D·§4-1·4-2 1차 — master_users.csv(98k행)·VALID 783, pandas+numpy(scipy 불요, Wilson/부트스트랩/BH 자체구현). 분 단위.
- BQ(조건부 1회): obs_lift 게이트. 분석창 2024-03~2026-03, `event_date` 파티션 필터 필수, APP 한정, `viewers/base=COUNT(DISTINCT user_id)`. POC가 5.57GB였으므로 단일 산출은 그 수준 — **단, §6-1·로컬 검정이 효과0을 가리키면 생략**.

### 6-3. 한계 (정직하게)
1. **조회 = '고려'지 '패시브 노출' 아님**(§3-3): `open_skill_description`은 상세 직접 열람이라 배너에 떠서 산 패시브 임프레션은 일부만 통제. "노출 결합" 중 *상세를 거친* 것만 약분됨. 순수 패시브 노출 편향은 본 신호로 완전 제거 불가 — 이건 데이터 한계로 명시.
2. **미커버 32%**(딥링크·마케팅 직구매·이력 우회): 조회 없이 산 전환은 분자에서 누락 → pair별 커버리지로 플래그하되 메우지 못함.
3. **볼륨 하락 추세**(54k→18k/일, §3-3): 최근 창일수록 obs 분모 얇아짐 → 신뢰구간 확대. 창 길이 민감도(7/14/30d)로 흡수.
4. **다른날 제외의 대가**: 인세션을 빼면 표본이 master 기준 재방문 23%로 줄어 일부 앵커가 pair 게이트 탈락 → 커버리지↓(§4-4와 trade-off). 이건 결함#3 제거의 *불가피한* 비용이며, 줄어든 표본에서도 신호가 살아야 진짜.
5. **인과 아님**: obs_lift>1이어도 "조회가 구매를 유발"이 아니라 "관심 있는 사람이 조회도 구매도"일 수 있음(역인과/공통원인). 본 신호는 *추천 랭킹용 연관*이지 처치효과가 아니며, 증분은 §action-plan의 CRM A/B로만 확정. 본 트랙은 그 A/B의 *후보 우선순위*를 댈 뿐.
