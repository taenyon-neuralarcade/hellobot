# Plan C — 행동시퀀스·사용자여정 기반 cross-sell 재설계

**작성**: 2026-06-09 · **렌즈**: 행동시퀀스/사용자여정 전문 · **목표**: 첫 구매자의 2번째 구매(1→2 전이, 현 31.5%) 유도
**대전제**: co-purchase 단일페어("A 다음 B 구매") 통계를 **버리지 않되 위계의 최하단으로 강등**한다. 그 위에 노출편향에 강건한 **다른 행동신호**(다른날 재방문·의도전이·여정단계)를 1차 신호로 올린다.
**근거 데이터**: `data/master_users.csv` (A 앱퍼스트 진짜 첫구매자 98,392명) 로컬 실측. BQ 불필요.

---

## 0. 한 줄 논지 — 왜 "다른 신호"인가 (두 시도의 공통 사인)

P0(구매 co-purchase)도 POC(노출정규화)도 **동일한 약점**으로 죽었다: 둘 다 **"A→B 단일 페어 통계"** 위에 서 있다.

- P0: f_menu→s_menu 페어. **68%가 당일(인세션)** → 페어가 "관심"이 아니라 "같은 세션 노출 결합"을 학습.
- POC: open_skill_description 게이트로 노출을 분모에 넣어 그 페어를 고치려 함 → **분자 90~95%가 또 당일**, 인기 통제 시 잔존신호 −0.002, SQL 버그·코호트 좌측절단으로 반증.

**진짜 원인은 "페어를 어떻게 정규화하느냐"가 아니라 "페어 그 자체"다.** SKU 페어는 (1) 인세션 노출 오염과 (2) 표본 희소가 **동시에 사는 자리**다. 이번 세션 실측이 이를 정량으로 못박았다:

> **SKU 페어 top-1은 자기 자신도 못 재현한다.** r_menu(재방문) top-1을 50/50 split 으로 검증하니 **48%만 양분 일치**(n≥100 앵커 75개 중 36개). 동전던지기 수준. 그런데 **같은 신호를 intent 레벨로 추상화하면 split Δ ≤ 1.3pp**(재회→재회 79.5% vs 79.0%). 노출오염과 노이즈가 같이 사는 SKU 자리를 떠나면 둘 다 사라진다.

따라서 Plan C 의 핵심 동작은 **"추상화 사다리 올리기 + 시간축(다른날) 분리"** 두 가지다. 이 둘이 노출편향과 표본노이즈를 *동시에* 깎는다.

## 1. 문제 재정의 — "의미있는 결과"를 여정 적합성으로 정의

기존 정의("A를 산 사람이 다음에 B를 가장 많이 샀다")는 **빈도 = 적합성**이라는 암묵 가정을 깔고 있고, 그 빈도가 노출에 오염됐다. 재정의:

> **의미있는 추천 = 방금 산 것이 사용자를 데려다 놓은 *여정상의 위치*에서, 자연스러운 *다음 한 걸음*인 SKU.**

이 정의는 4개의 검증 가능한 조건으로 분해된다 — 추천 후보 B 는 모두 통과해야 한다:

| # | 조건 | 측정 컬럼/신호 | 실패 시 |
|---|---|---|---|
| **J1. 방향성** | A *이후에* 관심이 생긴 것인가 (A와 동시 노출로 산 게 아니라) | **다른날(`days_to_rev>0`) 재방문** 또는 **의도전이 매트릭스** | 인세션 번들을 "다음 관심"으로 오인 |
| **J2. 비보유·비중복** | 이미 가진 것/방금 산 것의 재판매가 아닌가 | `same_menu`(3.1%)·`rev_same_menu`(1.9%) 제외 | "방금 산 걸 또" UX 붕괴 |
| **J3. 단계 적합** | 사용자가 선 *여정 단계*(획득미끼 vs 재방문엔진)에 톤이 맞나 | `f_topic`의 *역할* + 재방문 destiny | 1회성 미끼에 또 미끼 추천 → 이탈 |
| **J4. 판매가능·콘텐츠** | 실제로 팔 수 있고 콘텐츠 SKU인가 | P0 게이트(`is_open & !stop & buyers30>0` + blocklist) | 죽은 링크/충전상품 노출 |

J4 는 P0(`build_nbo_p0.py`)가 이미 해결했으므로 **그대로 상속**한다. Plan C 의 신규 기여는 **J1·J3**(노출편향·여정단계)이며, J2 는 값싼 필터로 흡수한다.

**왜 이 정의가 노출편향을 우회하나**: J1 이 "다른날/의도전이"를 요구하는 순간, 인세션 노출 결합은 정의상 후보가 될 수 없다. 노출 분모를 만들어 *사후에* 빼는 POC 방식(오염을 측정해서 빼기)이 아니라, **오염이 들어올 수 없는 신호만 입력으로 받는다**(오염을 처음부터 배제). 이게 POC 보다 단순하고 견고하다.

## 2. 신호 위계 — 4개 신호 + 각 노출편향 강건성

위에서 아래로 신뢰도 순. 상위 신호가 표본/안정성 게이트를 통과하면 채택, 미달이면 다음 신호로 fallback(§4).

```
       강건성↑ / 표본↓                                       강건성↓ / 표본↑
  ┌──────────────────────────────────────────────────────────────────────┐
  │ S1 의도전이(intent transition)  ← 1차. 가장 강건·가장 안정         │
  │ S2 다른날 재방문 SKU (r_menu)    ← 2차. 방향성 확보된 구체 SKU      │
  │ S3 여정단계 모델 (미끼→엔진)     ← 톤·타이밍 결정 (가로지르는 메타) │
  │ S4 고려시퀀스 (funnel 상세조회순)← 보강. 구매 전 신호 (BQ, 선택)   │
  │ ─────────────────────────────────────────────────────────────────── │
  │ [P0 co-purchase s_menu]          ← 최하단. 인세션 슬롯에만, 게이트 후 │
  └──────────────────────────────────────────────────────────────────────┘
```

> ⚠ 위계 순서 주의: **표본 크기**만 보면 S1(의도)·S3(여정)이 크고 S2(재방문 SKU)가 작다. 그러나 S2 는 "구체 SKU 를 곧장 추천"할 수 있어 *해상도*가 높다. 그래서 **S1 으로 방향(어떤 의도/주제)을 정하고, 그 안에서 S2 로 SKU 를 고르고, 미달이면 S3 로 내려가는** 결합을 쓴다(§4). "1차"는 신뢰의 1차이지 단독 사용이 아니다.

### S1 — 의도전이 (intent transition) · **1차 신호**

`f_intents → s_intents` 전이. 의도(재회·새로운만남·속마음·궁합·임신출산…)는 **사용자가 어떤 *질문*을 들고 왔는지**를 SKU 보다 한 단계 위에서 표현한다.

**실측 (master_users.csv, f_intents→s_intents 페어 22,285건):**

| f_intents | n | 최빈 s_intents | conf% | lift | Wilson LB% | split-half Δ |
|---|---|---|---|---|---|---|
| 재회 | 9,697 | **재회** | 79.3 | 1.82 | 78.5 | **0.5pp** |
| 새로운만남 | 3,898 | **새로운만남** | 49.7 | 3.19 | 48.1 | **0.4pp** |
| 속마음 | 2,860 | **속마음** | 36.6 | 2.58 | 34.9 | **0.6pp** |
| 궁합 | 1,819 | **궁합** | 42.9 | 6.01 | 40.6 | **1.2pp** |
| 썸짝사랑 | 1,428 | 썸짝사랑 | 40.5 | 4.29 | 38.0 | — |
| 임신출산 | 281 | **임신출산** | 80.8 | 65.7 | 75.8 | **1.3pp** |

**핵심 패턴 = 의도 *지속*(diagonal)**. 사용자는 "재회가 궁금해서" 첫 스킬을 샀고, 다음에도 재회가 궁금하다(79.3%). 이것이 연애 72% 재구매 엔진의 미시 메커니즘이다. lift 가 1.82~6.01 로 base 대비 명확히 높고, **Wilson 하한이 conf 에 거의 붙어 있다**(표본이 커서 불확실성 작음).

**노출편향 강건성 — 왜 S1 이 오염을 못 받나:**
1. **방향성 내재**: 의도전이는 "A 의도로 시작 → B 의도로 이동"의 *순서*를 매트릭스로 보존. 인세션 동시노출은 "동시"라 방향이 없음 → 매트릭스에 들어올 자리가 없음.
2. **추상화로 노출오염 희석**: 인세션 번들은 *특정 SKU 페어*에 꽂힌다(예: "속마음" 스킬 뒤에 "그사람 속마음" near-dup 깔기). 의도 레벨로 올리면 그 SKU 페어가 "속마음→속마음" 한 셀로 뭉개져 **개별 번들 배치의 흔적이 사라진다**.
3. **콜드스타트 면역**: 신상품 SKU 도 태그(`taenyon_temp_skill_tag_info_v2`)만 붙으면 즉시 의도 매트릭스에 편입 → co-purchase 이력 0 이어도 "재회 의도 신상품"으로 추천 가능. SKU 페어는 출시 후 n≥50 쌓일 때까지 추천 불가.

**한계**: ① **커버리지 37%** — `s_intents` 비어있는 63%(주로 비연애·태그 없는 SKU)엔 적용 불가 → S2/S3 로 fallback. ② **연애 편중** — intents 태그가 사실상 연애 도메인에 집중. 비연애는 §S3(topic)가 담당. ③ `r_intents` **컬럼 부재** — 재방문 의도전이는 직접 못 봄. s_intents(68% 당일 포함)로 학습하되, **diagonal(의도지속)만 채택**하면 당일/재방문이 같은 방향(§readme 로그 "동일주제 연속 68.8/67.9%, TV 0.025")이라 인세션 오염에 둔감(지속은 노출과 무관한 사용자 고유 관심).

### S2 — 다른날 재방문 SKU (r_menu) · **2차 신호**

`days_to_rev>0` 인 **다른날 첫 재방문 결제** SKU. §task2 §2 에서 *이미 반증을 견딘* 신호 — 인세션이 설명 못 하는 구매(사용자가 앱을 닫았다가 *관심으로 다시 와서* 산 것).

**실측:**
- 커버리지: **38.2%**(37,588명)가 다른날 재방문 결제 보유. 앵커 게이트 n≥50: **117 앵커가 페어의 85%** 커버. n≥30: 174 앵커 90.9%.
- **s_menu 와의 발산**: s·r 모두 n≥50 인 117 앵커에서 **top-1 이 48% 어긋남**, top-5 겹침 평균 **2.84/5(57%)**. → 재방문 추천을 s_menu(인세션)로 대신하면 절반의 앵커에서 1순위가 틀림.
- **타이밍**: days_to_rev **median 5일**(p25=1, p75=22). days_to_2nd median 0일(당일)과 완전히 다른 시간대 → **푸시/홈탭의 발화 타이밍 설계 근거**(첫구매 후 3~7일 윈도우).
- **보유필터 값쌈**: rev_same_menu(재방문에 같은 스킬 재구매) 1.9%, 재방문자 한정 5.0% → J2 필터 부담 미미.

**노출편향 강건성:**
1. **시간 분리 = 세션 분리**: 다른날 구매는 정의상 첫구매 세션의 노출 슬롯과 단절. 인세션 끼워팔기(P0 오염의 68%)가 **구조적으로 배제**된다. 이것이 §task2 §2 에서 "반증을 견딤"의 실체.
2. **잔여 오염은 푸시/리타게팅**: 다른날이라도 마케팅 푸시·리타겟 광고로 유도된 구매는 남음 → S2 단독 top-1 보다 **S1(의도) 일치 여부로 필터**(§4: S1 방향과 어긋나는 S2 SKU 는 강등).

**한계**: ① **불안정** — split-half top-1 일치 **48%**(SKU 레벨 노이즈). → **S2 를 단독 top-1 으로 쓰지 않는다.** S1 이 정한 의도/주제 *안에서* r_menu 빈도로 SKU 를 고르는 **conditional 용도**로만. ② 표본 38% → 재방문 없는 62%는 S3. ③ 인기 SKU 쏠림 → P0 의 lift 분모(P(Y as 2nd)) + Wilson 정렬 상속.

### S3 — 여정단계 모델 (획득미끼 → 재방문엔진) · **메타 신호 (톤·타이밍·fallback)**

첫 스킬의 *주제*가 사용자 여정에서 어떤 **역할**인지로 다음 추천의 톤을 결정. 리포트의 "신년/임신/결혼 = 고가 1회성 획득미끼" vs "연애/관계 = 재방문엔진"을 **f_topic→r_topic 매트릭스로 정량화**.

**실측 (f_topic → r_topic, 재방문 destiny):**

| f_topic (첫 주제) | users | 재방문율% | 당일2nd% | **→연애 전이%** | 역할 판정 |
|---|---|---|---|---|---|
| 연애 | 71,129 | 40.7 | 68.9 | **85** | **엔진(자기지속)** |
| 결혼 | 6,897 | 36.9 | 62.5 | **65** | **미끼→엔진** |
| 총운 | 5,075 | 32.5 | 56.2 | 49 | 미끼→엔진(약) |
| 자기탐구 | 905 | 31.7 | 74.0 | 47 | 미끼→엔진(약) |
| 일반운세 | 5,429 | 31.2 | 72.3 | 44 | 미끼→엔진(약) |
| 학업직업 | 3,984 | 29.2 | 70.7 | 26 | **니치(자기지속 36%)** |
| 재물금전 | 2,421 | 28.1 | 71.0 | 26 | 니치 |
| 가족자녀 | 2,058 | 21.6 | 59.8 | 14 | **니치(자기지속 49%)** |

**3가지 여정 구조가 데이터로 드러남:**
- **엔진(연애)**: 재방문 시 85% 연애 유지. 의도전이(S1)가 가장 잘 작동하는 영역. → S1·S2 1차 적용.
- **미끼→엔진(결혼·총운·자기탐구·일반운세)**: 첫 주제는 고가·1회성이지만 재방문 시 **연애로 49~65% 유입**. 사용자는 "결혼 운세"로 들어와 "연애" 사용자로 안착. → **다음 추천 톤 = 연애 엔진으로 핸드오프**. (결혼 first 의 연애 전이 65% 는 S1·S2 보다 강한 신호 — 미끼 앵커는 연애 우선 추천이 정답.)
- **니치(가족자녀·학업직업·재물금전)**: 연애로 안 감(14~26%). **자기 주제 내 지속**(가족자녀→가족자녀 49%, 학업직업→학업직업 36%)이 최빈. → 동일 주제 내 추천.

**타이밍 (여정단계별 재방문 시차):** 연애 median 5일 · 결혼 7일 · 가족자녀 11일. 미끼·니치일수록 재방문이 늦음 → **푸시 윈도우를 주제별로 차등**(엔진 3~7일, 니치 7~14일).

**형식(ctype) 핸드오프**: f_ctype→r_ctype 실측 — 사주→사주 64%·타로→타로 72%(두 기둥은 sticky), 진단→타로 67%·점성학→사주 54%(보조형식은 두 기둥으로 수렴). → **추천 형식 = 사주/타로 유지가 디폴트, 보조형식 첫구매자는 사주/타로로 변환**.

**노출편향 강건성:** 여정단계는 *주제 9종*의 거시 전이라 표본이 매우 크고(셀당 수백~수만), 개별 노출 배치가 통계에 미치는 영향이 무시할 수준. **노출오염에 가장 둔감**. 단, 해상도가 낮아(주제만) **단독으로는 SKU 를 못 고름** → S1/S2 의 *방향타*이자 *최종 fallback*.

**한계**: 주제 레벨이라 "어떤 연애 스킬"까지는 못 정함 → S1/S2 와 결합 필수. base-rate 가공물 위험(연애가 워낙 커서 "→연애"가 자동으로 큼) → **lift 로 보정**하고 "연애 default"를 맹신하지 않음(§5 검증).

### S4 — 고려시퀀스 (funnel 상세조회 순서) · **구매 전 신호 (BQ, 선택적 보강)**

`mart_v2_skill_funnel_fb` 의 `open_skill_description → enter_skill → click_start_button` **순서**와 `related_skill_seq`/`recommend_seq`. "샀지만 *다음에 볼* 것"을 구매 이전 단계에서 포착.

- **용도**: S1~S3 는 *구매 완료* 신호. S4 는 **구매 직전 고려(상세조회)**를 봐서, 아직 안 산 관심을 추천에 선반영(콜드 사용자·신규 세션).
- **노출편향 주의**: open_skill_description 은 "고려"라 패시브 임프레션보다 정밀하나(§task2 §3-3), **추천경유 조회(recommend_seq)는 그 자체가 노출 결과** → recommend_seq 경유 조회는 S4 입력에서 제외하고 **organic 경로(카테고리·검색) 조회만** 사용해야 노출오염 회피.
- **위치**: 본 설계의 **선택 트랙**. master_users.csv 로 검증 불가(BQ 필요), POC 가 이미 funnel 게이트의 함정을 보였으므로 **S1~S3 확정 후 보강용으로만** 착수. 무리한 선행 금지.

## 3. 지표 정의 (수식) + 로컬 실측 근거

모든 지표는 **방향 보존(정방향만)·표본 게이트·Wilson 하한 정렬**을 P0 에서 상속한다(과제4). 신규는 추상화 레벨(SKU→intent→topic).

### 3-1. 의도전이 신뢰도·lift (S1)

```
conf_intent(j | i) = N(f_intents=i ∧ s_intents=j) / N(f_intents=i)
lift_intent(j | i) = conf_intent(j|i) / P(s_intents=j)          # base = 전체 2nd 의 의도 분포
선정 = argmax_j  wilson_lb(conf_intent(j|i), N(f_intents=i))     # Wilson 하한 정렬
게이트: N(f_intents=i) ≥ 200  AND  pair_n ≥ 50  AND  lift ≥ 1.2
```
**실측 근거**: 재회·새로운만남·속마음·궁합 모두 게이트 통과(N 1,819~9,697, lift 1.82~6.01, wLB 34.9~78.5%). **split-half Δ ≤ 1.3pp** → 안정성 입증. (이 안정성이 S1 을 1차로 올리는 결정적 근거.)

### 3-2. 조건부 재방문 SKU (S2, S1 방향 내에서)

```
# S1 이 의도 j* 를 정함 → r_menu 후보를 j* 적합도로 필터 후 빈도 정렬
cand(X) = { b : f_menu=X 인 사용자의 r_menu=b, b∈VALID, b≠X, b 의 의도 ∈ {j*, 연애 default} }
conf_rev(b|X) = N(f_menu=X ∧ r_menu=b) / N(f_menu=X, r_menu≠null)
lift_rev(b|X) = conf_rev(b|X) / P(r_menu=b as 2nd)
선정 = argmax_b wilson_lb(conf_rev(b|X), N_rev(X))
게이트: N_rev(X) ≥ 50  AND  pair_n ≥ 10
```
**실측 근거**: n≥50 앵커 117개가 재방문 페어 85% 커버. **단, split top-1 48% 불안정** → 반드시 S1 의 의도 필터를 *먼저* 통과시켜 후보 공간을 좁힌 뒤 적용(단독 사용 금지).

### 3-3. 여정단계 전이·역할 점수 (S3)

```
role_score(t) =  P(r_topic=연애 | f_topic=t)        # 미끼→엔진 강도
                 ─ 높으면 "엔진 핸드오프", 낮으면 "니치 자기지속"
trans_topic(u | t) = N(f_topic=t ∧ r_topic=u) / N(f_topic=t, r_topic≠null)
lift_topic(u | t)  = trans_topic(u|t) / P(r_topic=u)   # base-rate 보정 (연애 과대 방지)
```
**실측 근거**: 결혼 role 0.65·총운 0.49 vs 가족자녀 0.14·학업직업 0.26 → 미끼/니치 이분이 데이터로 갈림. lift 보정 시 니치의 "자기주제 지속"(가족자녀→가족자녀 49%)이 연애 default 를 이김.

### 3-4. 안정성 지표 (모든 신호 공통 — 검증 게이트)

```
stability(signal, anchor) = 1[ top1(half_odd) == top1(half_even) ]   # 50/50 split
채택 조건: SKU 레벨은 stability 불충분 → intent/topic 레벨로 승격하거나 S1 방향 종속
```
**실측 근거 (이 설계의 척추):**

| 레벨 | 신호 | split-half top-1 일치 / Δ | 판정 |
|---|---|---|---|
| SKU 페어 | r_menu (n≥100) | **48% 일치** | ❌ 단독 불가 |
| intent diagonal | 재회/새만남/속마음/궁합/임신 | **Δ 0.4~1.3pp** | ✅ 1차 |
| topic 전이 | f_topic→r_topic | 셀당 수백~수만 | ✅ 메타 |

→ **추상화 레벨이 곧 안정성**. 신호 위계 = 안정성 위계.

## 4. 결합·fallback 전략 (콜드스타트·신상품 포함)

신호를 단일 점수로 섞지 않는다(POC 가 base_cvr 로 나눠 산술부작용을 낸 교훈). 대신 **계층적 게이트(cascade)**: 상위 신호가 *방향*을 정하고 하위 신호가 *해상도*를 채운다. 각 단계는 표본·안정성 게이트를 통과해야 다음으로 못 내려간다.

```
입력: 첫구매 스킬 X (f_menu), 그 주제 t(f_topic)·의도 i(f_intents)·형식 c(f_ctype), 표면(결제직후 | 재방문)

STEP 0  여정단계 판정 (S3) ─ 항상 먼저
        role = role_score(t)
        ├ role ≥ 0.45 또는 t=연애  → "엔진/핸드오프 모드": 추천 톤 = 연애, 형식 = 사주/타로 유지
        └ role < 0.45 (니치)        → "니치 모드": 추천 톤 = 동일 주제 t 내, 형식 유지
        타이밍 = {연애 3~7일 · 미끼 5~10일 · 니치 7~14일}  (재방문 표면일 때)

STEP 1  의도 방향 결정 (S1) ─ 연애/엔진 모드에서
        i 의 게이트 통과 시  j* = argmax wilson_lb(conf_intent(·|i))   # 보통 diagonal(의도지속)
        미통과(비연애·태그없음) → j* = None, STEP 3 로

STEP 2  SKU 해상도 (S2, j* 방향 내) ─ 표면별 신호 선택
        표면=재방문  → r_menu 후보를 j* 적합도로 필터 → wilson 정렬 top-N
        표면=결제직후 → s_menu(P0) 후보를 j* 적합도로 필터 → wilson 정렬 top-N   # 인세션 슬롯은 P0 유지하되 J1~J4 게이트
        N_rev(X)≥50 또는 pair_n≥10 미달 → STEP 3

STEP 3  주제 fallback (S3) ─ SKU 신호 부족 시
        t 의 r_topic 전이 top-1 주제 u* (lift 보정) 안에서 인기 SKU(VALID, Wilson)
        → SKU 콜드(82%)·신상품 모두 여기서 구제

STEP 4  글로벌 fallback ─ 그래도 비면
        nbo_fallback_popularity_a (전역 인기, VALID·blocklist 후) + 여정단계 톤 필터
```

**콜드스타트 처리 (3종):**
- **신규 사용자(첫구매 직후, 이력 0)**: STEP 0~1 만으로 충분 — 첫 스킬의 의도/주제만 알면 방향 결정. SKU 이력 불필요.
- **신상품 SKU 가 *첫구매*인 경우**: co-purchase 이력 없음 → STEP 0(주제)·STEP 1(의도 태그)로 우회. 태그만 있으면 즉시 추천 가능.
- **신상품 SKU 를 *추천 후보*로**: STEP 2 의 SKU 빈도엔 안 잡힘 → STEP 3 주제 fallback 에서 "해당 주제 신상품" 슬롯을 ε(예: 10%) 확보(과제 사소항목 "롱테일 소실" 대응, per-skill 노출캡 + ε-greedy).

**왜 이 cascade 가 두 시도보다 나은가:**
- 단일 점수 합산이 아니라 **방향(상위)→해상도(하위)** 분리 → POC 의 "인기로 나눠서 다 강등" 산술부작용 없음.
- 상위 신호(의도·주제)가 안정적(Δ≤1.3pp)이므로 **불안정한 SKU 신호를 안정적 방향 안에 가둠** → SKU 노이즈가 추천 *주제*를 흔들지 못함.
- 인세션 P0 는 죽이지 않고 **결제직후 슬롯의 STEP 2 후보 생성기로만** 강등 + J1~J4 게이트 → 표면 적합성 확보.

## 5. 검증 설계 + 한계

### 5-1. 오프라인 검증 (로컬, master_users.csv — BQ 불필요)

| # | 검증 | 방법 | 합격선 |
|---|---|---|---|
| V1 | **신호 안정성** | 각 신호 50/50 split top-1 일치율 (이미 실측: SKU 48% / intent Δ≤1.3pp) | intent·topic 레벨 ≥ 90% 또는 Δ≤2pp |
| V2 | **방향성(인세션 배제) 효과** | r_menu 추천 vs s_menu 추천의 top-1 발산(이미 48%) — 발산 큰 앵커에서 r_menu 가 "여정 다음"으로 더 타당한지 이름으로 정성검수 | 발산 앵커 ≥30%·정성 타당 |
| V3 | **여정단계 분류 타당성** | role_score 로 미끼/니치 분류 후, 미끼 앵커의 "연애 핸드오프" 추천이 실제 재방문 주제와 일치하는지 | 미끼 앵커 연애 전이 ≥45% 확인(✅ 결혼 65·총운 49) |
| V4 | **holdout 재현(time-split)** | 시간 분할 불가(컬럼에 첫구매 연도 f_year 만) → f_year 2024 학습 / 2025 검증으로 top-1 재현 | 주제·의도 레벨 top-1 안정 |
| V5 | **커버리지 누적** | STEP 1→2→3→4 각 단계가 커버하는 첫구매자 비율 (S1 37% + S2 38% + S3 주제 99.7% → 누적 100%) | 전 앵커 추천 산출 가능 |

> V1·V2·V3 는 이번 세션에서 *부분 실측 완료*(위 수치). 정식 검증은 cascade 구현 후 동일 스크립트로 일괄.

### 5-2. 온라인 검증 (출시 게이트 — 과제5 상속, 본 설계 SSOT 아님)

- **노출군 vs 비노출군 A/B**, 북극성 = 첫구매자→2번째 *증분* 전환율. 적중률은 보조 강등.
- **표면 분리 A/B**: 결제직후 슬롯(S2-인세션)과 재방문 푸시(S2-재방문)를 *별도 셀*로 — 두 표면이 48% 다른 추천을 내므로 통합 측정 금지.
- 1회성 이탈자(26.5%, 2번째 도달 안 함)는 학습 표본에 없음 → 이들엔 STEP 4 인기 fallback 분기(S1~S3 의 "관심" 신호는 2번째 도달자 취향이라 이탈자엔 과적합 위험).

### 5-3. 한계 (정직하게)

1. **`r_intents` 컬럼 부재** — 재방문 의도전이를 직접 못 본다. S1 은 s_intents(68% 당일 포함)로 학습하고 *diagonal(의도지속)만* 채택해 우회하나, 재방문 *전환*(의도가 바뀌는 케이스, 예 결혼시기→새로운만남 lift 2.64)은 sameday 신호로 근사 → BQ `mart_use_skill_se` 시퀀스로 재방문 의도전이를 직접 재구성하면 보강 가능(선택).
2. **S1 연애 편중** — intents 태그가 연애에 집중. 비연애 도메인은 S3(주제)만으로 SKU 해상도가 낮음. 비연애 의도 태깅 확대가 근본 개선(데이터 카탈로그 과업).
3. **S2 표본 불안정** — 다른날 재방문 38%·SKU split 48%. 단독 금지로 완화하나, 재방문 앵커가 적은 SKU 는 STEP 3 로 떨어져 해상도 손실.
4. **여정단계의 base-rate 위험** — 연애가 커서 "→연애"가 자동으로 큼. lift 보정으로 막지만, "연애 default"가 진짜 여정인지 base-rate 가공물인지 일부 앵커는 모호(과제 사소항목과 동일 주의).
5. **선택효과(2번째 도달자만)** — 모든 전이 신호는 *2번째를 산 사람*에게서 학습 → 안 산 사람(이탈자)엔 외삽. A/B 증분으로만 진짜 효과 확정(5-2).
6. **시즌물** — 신년/임신/결혼 시즌 SKU 는 작년→올해 SKU 치환 필요(전이 자체는 잡히나 SKU 가 교체됨). 시즌 트랙 별도(과제 사소항목).

## 6. 산출 예시 — 사용자 여정에서 추천이 어떻게 보이나

실측 수치로 구성한 3개 여정 시나리오(STEP cascade 적용 결과):

### 예시 A — 엔진 사용자 (연애·재회)
```
첫구매: "어떤 사람과 사귈 운명일까?" (f_topic=연애, f_intents=재회, f_ctype=사주)
STEP0  role(연애)=0.85 → 엔진 모드, 형식 사주 유지, 타이밍 3~7일
STEP1  S1: 재회→재회 conf 79.3%, lift 1.82, wLB 78.5% (Δ0.5pp 안정) → j*=재회
STEP2  [재방문 표면] r_menu 후보를 '재회' 의도로 필터 → wilson top-1
       추천: "그 사람의 속마음 / 다시 만날 수 있을까" (재회 클러스터 재방문 SKU)
       [결제직후 표면] s_menu(P0) '재회' 필터 → "지금 날 좋아하는 사람은?" (인세션, 게이트 통과)
→ 같은 첫구매라도 표면 따라 1순위 갈림(실측 48% 발산) — 의도(by 사주/재회)
```

### 예시 B — 미끼→엔진 핸드오프 (결혼)
```
첫구매: "2026 결혼운: 나의 결혼 시기는?" (f_topic=결혼, 고가 1회성 미끼)
STEP0  role(결혼)=0.65 → 핸드오프 모드 (재방문 65%가 연애로 이동), 타이밍 5~10일
STEP1  결혼 의도 태그 약함 → j*=연애 default
STEP3  결혼→연애 전이 안에서 인기 연애 SKU (lift 보정)
       추천: "올해 내 사랑은 어떻게 흘러갈까" — 결혼 미끼를 연애 엔진으로 안착
→ "결혼 운세 또 사세요"가 아니라(니치 아님) 연애 여정으로 핸드오프 (실측 65% 근거)
```

### 예시 C — 니치 사용자 (가족자녀) + 콜드 SKU fallback
```
첫구매: 신상품 "우리 아이 기질·진로" (f_topic=가족자녀, co-purchase 이력 0)
STEP0  role(가족자녀)=0.14 → 니치 모드 (연애로 안 감 14%, 자기지속 49%), 타이밍 7~14일
STEP1  의도 태그 없음 → j*=None
STEP2  신상품이라 SKU 빈도 없음 → 미달
STEP3  가족자녀→가족자녀 전이(49%) 내 인기 SKU + 신상품 ε-슬롯
```
