# 종합 — 실제 사용자에게 의미있는 결과를 내는 데이터·방법

**작성**: 2026-06-09 (코디네이터) · **입력**: 3개 독립 설계안 [`plan-A`](plan-A-causal-experimental.md)·[`plan-B`](plan-B-observational-signal.md)·[`plan-C`](plan-C-behavioral-sequence.md) · **배경**: P0(구매 co-purchase) + 노출정규화 POC **반증** ([`../task2-signal-correction.md`](../task2-signal-correction.md) §3-7)
**기준**: "어떤 데이터로 어떻게 봐야 *실제 사용자에게 의미있는* 결과가 나오나"

## 2. 세 안이 수렴한 것 (= 흔들리지 않는 척추)

독립적으로 작업했는데 **5가지에 동시 수렴**했다. 이게 가장 신뢰할 결론:

1. **결과(outcome) = 재방문(다른날) 2번째 구매. 당일(인세션) 제외.**
   - A: 북극성 = 재방문 증분(인세션 50%는 전환 직전이라 증분 작음). B: 분자·분모 다른날만. C: r_menu 시간분리.
   - → POC 반증의 핵심(분자 90% 당일)을 셋 다 정면 교정. **증분 여지·노출강건성·의미가 모두 재방문에 있다.**
2. **코호트 = 진짜 첫구매자**(master_users.csv, 2021 좌측가드, user_id 고정). POC의 좌측절단(앵커 40~48% 가짜) 제거.
3. **단일 비율/점수로 인기로 나누지 마라.** B: leave-out base+npmi. C: 단일점수 합산 금지·cascade. A: lift는 보조. (POC −0.23 = base_cvr 나눗셈 부작용의 교훈.)
4. **후보 = 판매중 유료 스킬(P0 VALID).** free/WEB결제 오염 제거.
5. **오프라인 신호는 프록시일 뿐, 진실은 실험(holdout).** 적중률 36.6%엔 대조군이 없다.

## 3. 분업 — 세 안은 경쟁이 아니라 **3개 층**

```
 [신호층 C] 무엇을 추천?      intent transition + 여정단계 cascade, r_menu로 SKU 해상도
      │   (SKU 페어 위로 추상화 → 82% 콜드스타트·노출오염·노이즈 동시 해결)
      ▼
 [게이트층 B] 그 신호가 진짜?   로컬 0-BQ로 인기통제 partial·plausibility·effect-0 판정
      │   (artifact면 여기서 걸러 P0로 폴백 — POC 낭비 반복 차단)
      ▼
 [진실층 A] 사용자를 움직였나?  영구 holdout + CRM A/B로 재방문 증분 측정
          (오프라인 신호가 정한 WHO/WHAT을, 증분으로만 판정)
```

핵심: **C가 후보를 만들고 → B가 그게 인기/노출 artifact가 아님을 싸게 검증하고 → A가 실제 증분을 입증한다.** 어느 하나로 끝나지 않는다.

## 4. 통합 권고 — 어떤 데이터로 / 어떻게 / 무엇이 의미있는 결과

### 어떤 데이터
| 역할 | 데이터 |
|---|---|
| 코호트 | `master_users.csv` 진짜 첫구매자 98,392 (좌측가드·APP·user_id 고정) |
| **결과(outcome)** | **`r_menu` = 다른날 재방문 2번째 구매** (당일 `second_sameday=1` 제외) |
| 신호(무엇 추천) | `f_intents→다음 intents`(의도전이) + `f_topic→r_topic`(여정단계), SKU 해상도는 `r_menu` conditional |
| 후보 | P0 VALID 유료·판매중 스킬 |
| 진실(증분) | 영구 holdout + Braze CRM 노출 로그 / 인앱 슬롯 노출 로그 |

### 어떻게
1. **추상화 레벨을 올려 신호를 본다 (C).** SKU 단일페어는 자기 자신도 못 재현(split-half 일치 48%)이고 82%가 콜드 → 페어 비율은 원천적으로 불안정. **intent/여정 레벨은 안정(재회→재회 split-half Δ≤1.3pp)**이고 노출오염이 추상화로 희석되며 신상품 콜드스타트에 면역. SKU는 그 방향 *안에서만* 채운다.
2. **인기·인세션·intent floor를 통제하고, 남는 신호만 믿는다 (B).** leave-out base(null=1.0 복원)·npmi(인기불변)·다른날限定·partial Spearman(인기통제). **로컬 0-BQ로 먼저** 돌려 effect-0면 5GB BQ 풀빌드 생략하고 P0로 폴백.
3. **증분은 holdout으로만 판정한다 (A).** 노출군 vs 비노출(holdout)의 재방문 2번째 구매율 차. 적중률·conf·lift는 전부 보조로 강등(효과 판정 권한 없음). "이미 살 사람"(total_pays·c_rev90·의도지속 고base)을 통제.

### 무엇이 "의미있는 결과"인가
> **노출군의 재방문 2번째 구매율 − holdout의 그것 = 증분(Δpp).** 통계적 연관이 아니라 *실제 행동 변화*. 그리고 그 증분이 "원래 살 사람"이 아니라 **개입으로 새로 산 사람**에서 나왔음을, 동일 base 정의·propensity 통제로 보증.

이 정의가 세 함정을 동시에 막는다: 적중률 착시(A holdout), 인기 artifact(B partial), 인세션 자기강화(다른날 outcome).

## 5. 실행 순서 — 싼 것 먼저, off-ramp 내장

| # | 단계 | 비용 | 판정/off-ramp |
|---|---|---|---|
| 0 | **영구 holdout 격리** (A) | 앱개발 0 | 모든 측정의 반사실 기준선. 지금 즉시. |
| 1 | **로컬 0-BQ 신호·게이트** (B+C) | 0 (master만) | intent전이·npmi·r_menu를 P0와 동일기반 비교 + partial(인기통제) + plausibility. **여기서 신호가 인기/P0를 못 이기면 → P0 채택, 노출 풀빌드 생략** |
| 2 | (1이 통과 시만) BQ 노출 obs_lift 보강 | ~5GB | B의 effect-0 기준 충족해야 빌드 |
| 3 | **재방문 환기 CRM A/B** (A) | 앱개발 0 | 결제 +3일(재방문 median 5일) 발송, 노출군 vs holdout, 무처치 vs 인기 active-control. **증분으로 GO/STOP** |
| 4 | (3 입증 시만) 인앱 결제직후/홈 슬롯 제품화 | 앱개발 | 증분 확정 후에만 |

검정력(A 실측): 재방문 baseline 38.2%, +3pp 탐지 4,174/팔 → 앵커 풀링+전수 첫구매자+CUPED+순차검정으로 6~10주 방향 판정.

## 6. 과잉투자 경계 (하지 말 것)

- **노출 obs_lift 풀빌드를 1단계 전에 하지 마라** — POC 낭비 반복. 로컬 effect-0 게이트 먼저.
- **SKU 단일페어 비율 모델 고도화 금지** — 페어 자리가 노이즈·오염의 원천. 추상화가 답.
- **demographic(성별·나이) 개인화 금지** — age-표준화 lift 변화 5.1%, ROI≈0 (기검증).
- **복잡 추천모델 금지** — 단순 co-purchase가 적중 최강(36.6). 손볼 건 모델이 아니라 *결과 정의(재방문)·검증(holdout)*.

## 7. 한 줄 결론

**"진짜 첫구매자가 *다른 날 다시 와서* 산 것(재방문 2번째 구매)을, SKU 페어가 아니라 *의도·여정* 레벨에서 보고, 인기·인세션을 통제해 남는 신호만 추리고, 그 효과는 holdout 증분으로만 판정한다."** — 세 안이 각각 신호·게이트·진실을 맡아 이 한 문장으로 수렴한다. 가장 싼 첫 수는 holdout 격리(즉시)와 로컬 0-BQ 게이트(BQ 0)이고, 그 결과가 노출 트랙을 빌드할지 P0로 충분할지를 데이터로 정한다.
