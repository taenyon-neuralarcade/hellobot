# Plan A — 인과·실험 관점 재설계: "증분 2번째 구매"를 1차 목표로

**작성**: 2026-06-09 · **렌즈**: 인과추론·실험설계 · **상태**: 설계 계획서 (구현 전, 무거운 산출 없음)
**목표**: 첫 구매자의 **2번째 구매(1→2 전이)를 *증분(incremental)*으로** 끌어올리는 추천을, 상관이 아니라 인과로 설계·검증한다.
**이 문서의 위치**: 두 번의 실패(P0 구매 co-purchase, 노출정규화 interest_lift POC) 이후의 재설계. 기존 산출물을 폐기하지 않되, "추천 신호"를 **목표가 아니라 증분의 프록시**로 강등하고, 증분 자체를 실험·준실험으로 측정하는 골격을 세운다.

> **근거 수치는 모두 로컬 `master_users.csv`(98,392 진짜 앱퍼스트 첫구매자, 2021 좌측절단 가드) 재계산.** BQ 무거운 쿼리 없음. 검정력·표본은 §3, 출처는 각 표 각주.

## 0. 두 실패에서 학습한 단 하나의 교훈

| 시도 | 무엇을 봤나 | 왜 실패했나 | 공통 결함 |
|---|---|---|---|
| **P0 구매 co-purchase** | 누가 A 사고 B 샀나 (conf·lift) | 68% 당일(인세션) = "관심" 아닌 "같은 세션 노출/번들" 학습. 자기강화·인기쏠림 | **증분이 아니라 적중을 봄** |
| **노출정규화 interest_lift POC** | B 노출 대비 B 전환을 A 이후/평균으로 비교 | −0.23이 인기(base_cvr) 통제 시 −0.002로 소멸(산술 부작용), 분자 90~95% 당일, 앵커 40~48%가 진짜 첫구매자 아님, intent floor, 불공정 비교 | **증분이 아니라 상관을 봄** |
| **리포트 적중률 36.6%** | backtest top-N hit | holdout/대조군 컬럼이 **0** → "이미 살 사람"을 맞힌 것일 수 있음 | **증분이 아니라 적중을 봄** |

**교훈**: 셋 다 *관측 상관/적중*을 정교화하는 데 매달렸고, **"추천을 보여준 것이 2번째 구매를 *추가로* 만들었는가"라는 인과 질문에 답한 적이 없다.** 정교한 오프라인 신호를 한 단계 더 정제해도 이 질문엔 못 답한다 — **반사실(counterfactual)이 데이터에 없기 때문.** 따라서 재설계의 1순위는 더 나은 신호가 아니라 **반사실을 만드는 장치(holdout)**다.

> **과잉투자 경계 (이미 검증된 결론 계승)**: demographic 개인화(age-표준화 변화 5.1%)·복잡 추천 모델은 ROI≈0. 이 계획은 모델을 더 똑똑하게 만들지 않는다 — **이미 가진 단순 신호(P0)를 *증분 측정 루프*에 얹는 것**이 전부다.

## 1. 문제 재정의 — "사용자에게 의미있는 결과" = 증분 2번째 구매

### 1-1. 정의

추천이 "의미있다"는 것은 **추천을 본 사용자 집단이, 동일하지만 추천을 안 본 집단보다, 2번째 구매를 더 많이 했다**는 뜻이다. 수식으로:

```
의미있는 결과 = ATE = E[2번째구매 | do(추천노출)] − E[2번째구매 | do(노출안함)]
```

`do(·)`는 개입(intervention)이다. 관측에서 본 `P(2nd | 추천을 본 사람)`은 do가 아니라 조건부 — "추천을 본 사람"은 애초에 더 활동적이라 2번째를 살 사람이 모인 것일 수 있다(self-selection). **적중률·conf·lift·interest_lift는 전부 조건부지 do가 아니다.** 그래서 보조 지표로 강등한다.

### 1-2. 어디에 증분 여지가 있나 — 인세션 vs 재방문 분해 (실측)

1→2 전이를 타이밍으로 쪼개면 증분 여지가 극명하게 갈린다:

| 마진 | 도달률(첫구매자 대비) | 인과적 성격 | 증분 여지 |
|---|---|---|---|
| **인세션(당일) 2번째** | **50.0%** | 이미 결제 플로우 안, 전환 직전. 추천이 "관심"보다 "노출"로 작동(P0 68% 당일의 정체) | **작다** — 이미 살 사람이 대부분. 추천을 더 깔아도 신규 증분 한계. **자기강화 위험 최대** |
| **재방문(다른 날) 2번째** | **38.2%** | 세션을 떠난 뒤 *다시 돌아와* 산다. 돌아오게 만드는 외력(푸시/관심환기)이 인과적으로 개입할 빈 자리 | **크다** — "안 돌아올 뻔한 사람을 돌아오게" = 진짜 증분이 생기는 곳 |

> 출처: `master_users.csv` 재계산 (2026-06-09). ANY 2nd=73.5%, in-session=50.0%(`has_2nd&second_sameday`), revisit=38.2%(`r_menu` notna). 재방문 timing median 5일: 1일 25.7%·2~7일 30.6%·8~30일 24.9%·31~90일 18.8%.

**핵심 결정**: **북극성과 첫 실험은 재방문(other-day) 마진을 겨냥한다.** 인세션은 (a) 증분 여지가 작고 (b) 앱 개발이 필요하며 (c) 자기강화 위험이 커서, 인과 측정의 *후순위*다. 인세션 슬롯(기존 액션안의 P2)은 재방문 실험으로 방법이 검증된 뒤에 붙인다.

### 1-3. 추천 신호의 새 역할

P0 NBO(또는 그 후속)는 **버리지 않는다.** 다만 역할이 바뀐다:

```
[기존] 오프라인 신호(conf·lift) = 최종 산출물, 적중률로 자기검증
[재설계] 오프라인 신호 = "누구에게 무엇을 보여줄지"의 정책(policy)일 뿐.
        그 정책이 만든 증분은 오직 holdout 실험이 측정. 신호↔증분은 §5 보정 루프로 연결.
```

## 2. 지표 체계 — 북극성·가드레일·보조

### 2-1. 북극성 (1차, 단 하나)

```
NSM = 증분 재방문 2번째 구매 전환율
    = P(2nd_revisit | 노출군) − P(2nd_revisit | holdout)
```

- **단위**: 첫구매자(앵커 1명) = 1 관측. 기간: 첫 결제 후 **30일**(주 윈도우) · 60/90일 병행(민감도).
- **2nd_revisit 정의**: 첫 결제일 *이후 다른 날*의 유료(`pay_for_%`, `revenue_krw>0`) 결제 1건 이상. 같은 날(인세션) 2번째는 **분자에서 제외** — 노출 개입이 작동할 수 없는 구간이라 효과를 희석한다. (POC 실패 원인 ②"분자 90% 당일"의 정면 교정.)
- **추정량**: 무작위 배정이면 단순 차분(diff-in-means) + 2-비율 z검정. 공변량 보정은 CUPED(사전기간 공변량으로 분산 축소)만 — 효과 추정엔 무작위화가 식별을 보장하고 CUPED는 검정력만 올린다.

### 2-2. 가드레일 (악화하면 출시 중단)

| 가드레일 | 정의 | 우려 시나리오 | 임계 |
|---|---|---|---|
| **유저당 매출(ARPU)** | 첫 결제 후 30/90일 누적 `revenue_krw`/유저 | 싼 SKU로 갈아타게 만들어 2nd 건수↑ 매출↓(cannibalization) | 노출군 ARPU < holdout − 2% 면 경보 |
| **3번째까지 도달(2→3)** | 2번째 도달자 중 3번째 도달률 | 부적합 추천이 2번째를 "실망 구매"로 만들어 장기 이탈 | 노출군 2→3 < holdout − 2pp 경보 |
| **이탈/불만 프록시** | 푸시 수신거부율·언인스톨·CS 문의(채널별) | 과다 푸시·부적합 추천의 신뢰 훼손 | 수신거부율 노출군 > holdout + 0.5pp 경보 |
| **추천 위생(pre-ship)** | rank-1 중 판매불가·비콘텐츠·자기추천 비율 | P0가 잡은 24~35% 오염이 푸시로 회수불가 노출 | **0% 강제** (산출+서빙 하드게이트, action-plan §3.1 계승) |

### 2-3. 보조 (진단·정책개선용, 의사결정 권한 없음)

| 보조지표 | 용도 | **명시적 강등 이유** |
|---|---|---|
| CTR / 슬롯경유 전환 | 크리에이티브·슬롯 위치 진단 | 노출량·위치에 좌우, 증분 아님 |
| 적중률(backtest hit) | 정책(신호) 후보 A vs B 사전 비교 | holdout 없음 → "이미 살 사람" 적중(리포트 36.6%의 함정) |
| conf·lift·interest_lift | 정책 후보 *랭킹*에만 | 전부 조건부 상관. interest_lift는 POC에서 인기 통제 시 소멸 검증됨 |
| 노출 커버리지 | 정책 적용 가능 앵커 범위 | 실현가능성 지표지 효과 아님 |

> **원칙**: 보조지표는 "어떤 정책을 실험에 태울지"는 정해도, "그 정책이 효과 있었는지"는 절대 판정하지 않는다. 효과 판정 권한은 §2-1 NSM에만 있다.

## 3. 실험 설계 — 저비용 순 (앱 개발 0 → 0 → 제품화)

### 실험 0 — Always-on Holdout (모든 것의 전제, 즉시)

무엇을 하기 전에 **CRM 도달 가능 첫구매자의 5~10%를 영구 무작위 holdout으로 격리**한다. 이들에겐 어떤 cross-sell 푸시/카톡도 보내지 않는다. 이것이 **모든 후속 측정의 반사실 기준선**이자, "지금 우리가 보내는 모든 CRM이 합쳐서 증분을 내고 있나"를 처음으로 답하는 장치다(현재는 holdout이 없어 답 불가 — 리포트 적중률의 근본 한계).

- **무작위화 단위**: **유저(첫구매자)**. 앵커가 아니라 유저 — 한 유저가 여러 추천을 받을 수 있고, 이탈/매출 가드레일이 유저 단위라서. user_id 해시 기반 안정적 버킷(재계산에도 같은 유저는 같은 군).
- **개발량**: 0 (CRM 세그먼트 분기만). **이미 보내는 푸시가 있다면 오늘부터 holdout만 떼면 됨.**

### 실험 1 — 재방문 환기 CRM A/B (1차 핵심, 앱 개발 0)

**가설**: 첫 결제 후 *재방문 전 구간*에 "다음 관심 운세"를 푸시/카톡으로 환기하면, 안 보낸 군보다 재방문 2번째 구매가 증분한다.

| 설계 요소 | 결정 | 근거 |
|---|---|---|
| **무작위화 단위** | 유저(첫구매자). 앵커별 정책은 유저에 매핑 | 가드레일·중복노출 통제, 위 실험0과 동일 버킷 체계 |
| **개입(treatment)** | 첫구매 anchor의 **P0 정제 NBO top-1**(재방문 풀 `r_menu` 기반, 판매상태·blocklist·자기제외 하드게이트 통과)을 1회 푸시/카톡 | 신호는 정책일 뿐(§1-3). 재방문 표면이므로 `r_menu` 풀 사용(인세션 `s_menu` 아님 — 표면분리 §2 계승) |
| **대조(control)** | 무처치(no-send) **또는** 능동대조(인기순 fallback 발송) 2팔 | 무처치=총 증분(노출+선택 합), 능동대조=**추천 선택의 순효과**(노출량 동일) 분리. 둘 다 두면 "푸시 자체 효과" vs "개인화 선택 효과"를 가른다 |
| **노출 타이밍** | **재방문 전**: 첫 결제 +D일에 발송, 아직 다른날 2번째 안 한 유저만. D 후보 = **3일**(주력) · 1일·7일(arm) | 재방문 median 5일, 2~7일 구간에 30.6% 몰림 → 그 직전(3일)이 환기 최적창. 결제 직후(0일)는 인세션과 겹쳐 증분 희석 |
| **채널** | Braze 푸시 + 카톡채널(현 가용). 인앱 슬롯은 실험 2 | 앱 코드 0 |
| **표본/검정력** | 아래 표 | `master` 실측 baseline 38.2% |
| **기간** | 표본 충족까지 누적(throughput 기반, 아래) + 최소 2주 노벨티 흡수 | |

#### 검정력·표본 (two-proportion z, α=0.05 two-sided, power=0.80)

재방문 2번째 baseline = **38.2%**(실측). 탐지하려는 절대효과(MDE)별 팔당 N:

| MDE(절대) | 상대 | **팔당 N** | 2팔 합 | 판단 |
|---|---|---|---|---|
| +1.0pp | 2.6% | 37,239 | 74,478 | 너무 작은 효과 — 비현실적 N |
| **+2.0pp** | **5.2%** | **9,352** | **18,704** | **현실적 주력 타깃** (power 0.90이면 팔당 12,519) |
| +3.0pp | 7.9% | 4,174 | 8,349 | 빠른 1차 판정 |
| +5.0pp | 13.1% | 1,514 | 3,029 | 강효과만 잡음(under-powered 위험) |

> 출처: `/tmp` 로컬 검정력 계산(scipy 없이 정규근사, z_{α/2}=1.96, z_β=0.842). baseline은 master 실측.

**중요 주의 — 위 N은 "분모를 누구로 잡느냐"에 민감하다.** 38.2%는 *결국 재방문한 비율*(장기). CRM이 실제 미는 건 *발송 시점에 아직 재방문 안 한 유저의 그 후 전환*이라 **base가 더 낮을 수 있다**(8~12% 추정 시 +1.5~2pp 탐지에 팔당 5,500~7,800). → **분모를 "발송 시점 위험집합(at-risk)"으로 사전 정의**하고(§4 위험집합과 동일 정의로 통일), 파일럿 1주로 base를 실측해 N을 재고정한다.

#### Throughput·기간 sanity

`master` 98,392명이 ~24개월 창이라면 **신규 진짜 첫구매자 ~950/주**. 이 중 재방문-eligible(당일 2번째 없음) ~50% = **~475/주**가 실험 1의 모집단. → +2pp(팔당 9,352, 2팔 18,704) 충족엔 **약 39주**. **너무 느리다.**

**대응 (검정력 확보 전략, 우선순위):**
1. **앵커 풀링**: 유저단위 무작위화는 유지하되, *모든 앵커의 정책을 동시에* 한 트리트먼트 군에 적용(앵커별 따로 안 봄) → 모집단이 475/주 전체. 앵커별 효과는 사후 부분집단(고볼륨 앵커만).
2. **MDE 완화 + 순차검정**: 1차는 +3pp(팔당 4,174 → ~18주)로 빠르게 방향 판정, alpha-spending(예: Pocock/OBF) 중간분석으로 조기중단 허용.
3. **전수 첫구매자로 확장**: A 앱퍼스트뿐 아니라 앱 첫구매 전체로 모집단 확대(B 웹→앱 이동자 포함, 채널은 공변량/층화로 통제). 모집단 2~3배.
4. **CUPED**: 사전기간(첫 결제 전 활동·total_pays 분위)로 분산 축소 → 동일 N에서 검정력↑(보통 유효 N 1.3~2배 효과).

> 권고 운영점: **(앵커 풀링 + 전수 첫구매자 + CUPED)로 모집단·검정력을 키우고, 1차 MDE +3pp 순차검정으로 ~6~10주 내 방향 판정 → 유의하면 +2pp 정밀 추정으로 연장.**

### 실험 2 — 인앱 결제직후 슬롯 (제품화, 서버+앱)

실험 1에서 *재방문 환기의 증분이 입증된 뒤에만* 착수. 인세션 슬롯은 증분 여지가 작으므로(§1-2) **목표를 달리**: 인세션은 "전환율↑"이 아니라 **"객단가/2nd 품질↑·재방문 추가 유도"**로. 설계:

- **무작위화**: 유저 버킷(실험0 계승). 슬롯 노출 on/off.
- **개입**: 결제완료/결과 화면에 정제 NBO(인세션 풀 `s_menu` 하드게이트 통과) top-3.
- **1차 지표**: (a) 인세션 2nd 증분(작을 것으로 예상) + (b) **30일 재방문 2nd 증분**(슬롯이 "다음 관심"을 각인시켜 재방문을 늘리는가 — 더 중요) + (c) ARPU 가드레일.
- **개발량**: 서버(서빙 API·실시간 판매상태) + iOS/AOS(슬롯 UI·노출/클릭 이벤트) + design. → 별도 후속 프로젝트.

> **이벤트 부수효과**: 실험 2의 노출/클릭 이벤트는 §5 보정 루프와 근본 트랙(노출로그)의 입력이 된다 — 슬롯을 깔면 비로소 "노출했는데 안 산" 분모가 깨끗이 생긴다(현 funnel `open_skill_description`의 32% 사각 보완).

## 4. 관측 데이터로 증분 근사 — 실험 전/보완 (준실험)

실험은 미래 코호트가 쌓여야 답이 나온다. 그 전에 **기존 master로 증분의 *방향과 상한*을 근사**하되, **이것이 인과가 아님을 명시**하고 실험으로 보정한다(§5).

### 4-1. 이 데이터에 맞는 방법 = Exposure-controlled propensity + 위험집합 정의

POC가 시도한 노출정규화의 *의도는 옳았다* — 실패는 구현(viewers 버그·당일 미제거·좌측절단·불공정 base)이었다. 여기서는 **방법을 propensity 기반 증분 근사**로 바꾸고, POC 필수수정 6종을 설계에 내장한다.

**대상 질문(준실험)**: "첫구매 직후 *재방문 전 구간*에 추천 SKU B에 노출된(=funnel에서 B 상세조회) 유저가, 동일 성향인데 노출 안 된 유저보다 B를 더 샀는가?"

**식별 전략 — 위험집합(risk set) + 성향점수 매칭/가중:**

```
1. 위험집합 정의: 첫구매(A) 후, 아직 재방문 2번째를 안 한 유저 = at-risk.
   (NSM·실험1 분모와 동일 정의로 통일 — 분모 불일치가 POC 불공정 비교의 원인이었음)
2. 처치 = 위험집합 안에서 B를 상세조회(open_skill_description)한 사건.
   ⚠ 당일/동일세션 노출은 제외 — 다른-세션 노출만(POC ②교정).
3. 성향점수 e(X) = P(B 노출 | X). 공변량 X:
   - 재구매 성향: total_pays(사전), c_rev90/c_any90, days_to_rev 분위
   - 앵커: f_menu(A), f_topic, f_ctype, f_intents
   - 인구: age (보조, 효과 작음 검증됨)
   - 시간: 첫구매 코호트 월(시즌·볼륨 추세 통제 — funnel 54k→18k 하락 흡수)
4. 매칭/IPW로 노출군 vs 비노출군 균형 후 B-구매율 차분 = 노출의 근사 ATE.
5. "이미 살 사람" 통제: 위 X의 total_pays·c_rev90이 핵심 통제축.
   (잔차 confounding은 남음 — §4-3 한계)
```

**왜 propensity인가(이 데이터 적합성):** master에 재구매 성향 프록시(total_pays·c_rev*)와 앵커·코호트가 다 있어 X가 풍부하다. DiD는 노출 시점이 유저마다 흩어져 평행추세 가정이 약하고, 단순 RDD는 임계가 없다. **풍부한 사전 공변량 + 명확한 위험집합 = 매칭/IPW가 최적.**

### 4-2. 인세션 vs 재방문 분리 — 증분 여지 진단 (POC ⑤ 교정)

POC는 분자의 90~95%가 당일이라 "인세션 오염을 재세탁"했다. 여기서는 **두 구간을 분리해 각각 증분을 근사**:

- **인세션 구간**: 같은 세션 노출→같은날 구매. propensity로 통제해도 **노출과 구매가 거의 동시 = 증분 작고 식별 약함**. → "여기엔 증분 여지가 작다"는 것을 *수치로 보이는 용도*. 정책 결정엔 약하게만 반영.
- **재방문 구간**: 첫구매 후 다른날 노출→그 후 구매. **노출과 구매 사이 시간·이탈위험이 있어 개입 여지 존재**. → **여기서 나온 근사 증분이 실험 1의 사전 효과크기(MDE 가설)를 정한다.**

### 4-3. 명시적 한계 (이걸 안 적으면 POC를 반복)

| 한계 | 내용 | 완화 |
|---|---|---|
| **잔차 교란** | 매칭해도 "관심 있어 상세조회"라는 미관측 동기가 노출·구매를 동시 견인 → ATE 상향편의 | 노출=고려신호라 *상한*으로 해석. 실험이 진짜 ATE |
| **노출 소스 사각 32%** | `open_skill_description`이 딥링크·마케팅직구매·패시브배너 미포착 | pair별 커버리지<50% 플래그(task2 §3-6 계승), 결론 보류 |
| **좌측절단** | 앵커=진짜 첫구매 보장 필요(POC 40~48% 오염) | master의 2021 좌측가드 + 분석창 이전 무결제 조건 강제(POC ②) |
| **선택편의(base)** | base를 전체 아닌 동일 위험집합·분자유저 leave-out으로(POC ④) | null=1.0 복원, intent floor 제거 |
| **생존편의** | free/WEB 결제 스킬 base=0 탈락(POC ③) | 후보 B=판매중 유료(P0 VALID 재사용) |

> **판정 규칙(과잉투자 경계)**: 위 6종 교정 후에도 재방문 구간 근사 증분이 **0에 수렴하면** → 노출정규화·복잡 신호는 효과 없음. **P0(Wilson 정렬)을 정책으로 쓰고 실험 1로 직접 증분만 검증**한다. 근사가 양(+)이면 그 크기를 실험 1 MDE로 채택. **근사는 GO/NO-GO가 아니라 실험의 사전분포를 정하는 도구.**

## 5. 오프라인 신호 ↔ 실험 보정 루프

신호(정책)와 증분(진실)을 **닫힌 루프**로 연결한다. 신호는 영원히 프록시고, 실험이 주기적으로 보정한다.

```
        ┌──────────────────────────────────────────────────────────┐
        │                                                          │
        ▼                                                          │
[1] 오프라인 신호로 정책 후보 생성                                    │
    P0 NBO(재방문 r_menu 풀) → 앵커별 top-1. 하드게이트.              │
    §4 propensity 근사로 "증분 여지 큰" 앵커 가중.                    │
        │                                                          │
        ▼                                                          │
[2] 정책을 실험 1에 태움 (유저 무작위, holdout 대조)                  │
        │                                                          │
        ▼                                                          │
[3] 증분(NSM) 측정 — 앵커별/풀링                                     │
        │                                                          │
        ▼                                                          │
[4] 보정: 신호 순위 vs 실측 증분 순위 비교                            │
    - 일치 → 그 신호족을 정책으로 승격                               │
    - 불일치(고-conf인데 증분 0) → 자기강화/노출편향 의심,            │
      해당 pair 강등 + §4 propensity로 원인 진단                     │
    - 증분 큰데 신호 낮음 → "숨은 관심" 후보, 다음 라운드 노출 확대    │
        │                                                          │
        └─────────── 정책 갱신 (월 1회, 시즌기 주 1회) ─────────────┘
```

- **신호의 좌천 확정**: conf·lift·interest_lift는 [1] *후보 생성*과 [4] *순위 비교*에만 등장. **증분 판정은 항상 [3] 실험.** 신호가 증분과 어긋나면 신호를 의심하지 실험을 의심하지 않는다(POC의 반대 자세).
- **온라인 학습 경계**: 슬롯 노출 로그가 쌓이면(실험 2 이후) "노출 대비 전환"의 *깨끗한* 분모가 생겨, interest_lift를 *제대로* 재계산 가능 → 그때 근본 트랙 재개. 그 전엔 **실험이 신호를 대신**한다.
- **자기강화 차단의 구조적 보장**: 정책이 holdout을 영구 보유하므로, 정책이 만든 노출이 다시 정책을 강화해도 **holdout과의 차분은 항상 순수 증분**을 준다(closed-loop가 측정을 오염시키지 못함).
