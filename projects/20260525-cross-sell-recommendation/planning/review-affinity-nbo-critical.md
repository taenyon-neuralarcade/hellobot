## 🎯 한 줄 헤드라인

> **현재 affinity는 "X 산 사람의 관심"이 아니라 상당 부분 "앱이 과거에 인세션에 노출·번들·할인한 것"을 학습한다.** 학습 풀의 **68%가 당일(same-day) 구매**이고, 추천 출력의 **24%가 할인·충전·추가구매 SKU**, **24~35%가 판매불가 상품**이다. 이걸로 추천하면 과거 배치를 자기강화(closed loop)하고, 재방문 채널엔 인세션 업셀물을 민다. → **현 산출물은 "탐색적 참고"까지. 적용 전 아래 수정 필수.**

5개 에이전트가 독립적으로 수렴한 결함: ① same-day 68% 오염 (1·3·4·5) ② 머천/할인 SKU 침투 (2·3·4·5) ③ 판매불가 추천 (4·5) ④ 양방향 pooling 방향오류 (1·2·3) ⑤ small-n 무의미 (1·2·4·5) ⑥ 모집단 불일치 (1·2·4).

## 우선순위 1 — 즉시 제품 사고 (액션하면 바로 깨짐)

### P1. 판매불가·죽은 시즌상품을 추천 (rank-1의 24~35%)
- **수렴**: 에이전트 4(S1)·5(F1·F2)
- **무엇이 깨지나**: 결제직후 슬롯/푸시에 "다음 이거" 띄웠는데 탭하면 **구매 불가**. 카카오 푸시는 회수 불가라 더 치명적.
- **근거**: rank-1 234건 중 `is_stop_selling`/`is_open=false`/카탈로그부재 = **60~61건(26%)**, do-not-ship 합집합 **82건(35%)**. **'을사년'(2025) 추천 14건 전부 판매중지.** 학습창(2024-03~2026-03)이 현재(2026-06)와 단절 — 추천 등장 스킬 258개 중 open_date 2026+ 는 단 3개.
- **수정**: 산출 마지막에 `is_open=true AND is_stop_selling=false AND buyers_30d>0` **하드 게이트**. 시즌상품은 작년→올해 SKU 치환 룰. fallback에도 동일 게이트.

### P2. 할인·충전·추가구매 SKU가 "관심 추천" 1순위로 침투 (rank-1의 24%)
- **수렴**: 에이전트 2(F7)·3(F2)·4(S5)·5(F1·F9)
- **무엇이 깨지나**: "관심 콘텐츠 추천" 자리에 `🔋질문권 충전`·`(추천할인)…`·`[추가구매]…`가 노출. cross-sell 아님 + 측정 오염(슬롯 전환이 "추천 적중"인지 "할인 충동"인지 구분 불가).
- **근거**: 머천/업셀 SKU가 **53개 앵커(23%) top-5**에 ≥1개, **rank-1의 24%**. web-hero: 55723 rank-1 = 질문권충전(lift 31.97), 57670 rank-7 = `[추가구매] 병오년`(lift 240.99, 표 전체 최고가 동일상품 애드온).
- **수정**: 비콘텐츠(`topic=기타`/`ctype=null`)·업셀(`추천할인`·`추가구매`·`충전`·`패키지`·`프리패스`) menu_seq **hard-exclude**. action-priorities의 역할 태깅(획득미끼/재방문엔진/중립)을 추천 *전*에 적용.

## 우선순위 2 — 핵심 해석·방법 결함 (추천 자체가 틀림)

### P3. affinity = '관심'이 아니라 '인세션 노출/번들' 학습 (68% same-day)
- **수렴**: 에이전트 1(F3)·3(F1)·4(S3)·5(F4) — **가장 깊은 발견**
- **무엇이 깨지나**: 학습 풀(`has_2nd==1`)의 **68.1%가 당일 동시결제**. 당일 묶음은 "관심 전이"가 아니라 "한 세션 번들/업셀". ① 결제직후 NBO 슬롯은 *기존 인세션 추천 surface의 출력*을 다시 학습 → **과거 배치 자기강화(closed loop)**. ② 재방문 푸시/홈탭(2순위 액션)에 인세션 업셀물을 미는 미스매치 — 당일 vs 재방문 top5 SKU 중첩 **48%**(절반이 다름).
- **근거**: 2번째 72,300건 중 sameday 49,235(68.1%). 2141(속마음) 82% sameday. 머천 SKU 당일 6.2% vs 재방문 1.3%(5배 집중 = 노출 흔적). **baseline R3-0 "SKU·형식·재구매율은 진짜재방문(r_*)으로 분리" 계약 위반.**
- **수정**: 표면별 분리 산출 — 결제직후 슬롯 = `s_menu`(당일) 풀, **푸시/홈탭 재방문 = `r_menu`(진짜재방문) 풀**. 근본해법 = 노출(impression) 로그 기반 lift(현재 co-purchase에 노출 분모 없음).

### P4. 양방향 pooling이 추천 시간방향을 오염 (top-1 절반이 바뀜)
- **수렴**: 에이전트 1(F4)·2(F1)·3(F4)
- **무엇이 깨지나**: 추천은 "X 산 *뒤* Y"인데 코드(L30)가 `f==X의 s`(정방향)에 `s==X의 f`(역방향: Y 먼저)를 합산. 역방향은 추천 시점에 못 쓰는 순서.
- **근거**: 정방향-only vs pooled top-1이 **7개 앵커 중 5개에서 다름**(49816 400→28894 등). 앵커별 역방향 최대 79%. **앵커 4개(60706·60013·60244·59089)는 정방향 표본 0~2** = 추천표 전체가 역방향 데이터.
- **수정**: NBO는 **정방향만**(`pairs[f==X].s`). 순서무관 "동시관심"은 별도 용도로 명시 분리. 결과에 `fwd_share` 컬럼.

### P5. small-n 노이즈 + 통계 유의성 부재 (CSV에 게이트 없음)
- **수렴**: 에이전트 1(F5)·2(F2·F3·F6)·4(S6)·5(F3)
- **무엇이 깨지나**: nX 4~8 앵커(웹 3종)는 **n=1 추천**이 conf 33%·lift 40~241로 강신호처럼 보임. 코드의 `nX<50 ⚠`는 **print에만, CSV엔 전 행 기록** → 다운스트림이 무게이트 소비.
- **근거**: web-hero 추천행 16%가 nX<10 앵커. **12개 중 10개 rank-1이 rank-2와 Wilson 95% CI 겹침**(통계적으로 1위 아님). NBO도 85% pair_n<30, 60% <10, median 7. 2,003개 셀 다중비교 무통제(n=1이 lift 241).
- **수정**: `pair_n≥10`(권장 30) + `anchor_n2≥50` **CSV 산출단계 강제**, Wilson 하한 정렬, rank1-rank2 CI 분리 시만 "추천". 다중비교 보정(BH-FDR).

### P6. 모집단 불일치 — 앱 학습 → 웹 적용 + "앱=깨끗" 전제 도전
- **수렴**: 에이전트 1(F1)·2(F11)·4(S2·S4)
- **무엇이 깨지나**: 타겟='웹 X 구매자'인데 학습='A 앱퍼스트'. 두 집단의 진입스킬·연령·재구매 메커니즘이 다름 → 앱 룰을 웹 사용자에 적용 시 존재 않는 행동에 베팅.
- **근거**: 첫구매 지지율 역전(2141 앱 4.13% vs 웹 0.67%; 49816 웹 3.18% vs 앱 0.80%). 연령 `<25` **A 38.7% vs C 18.2%**(2배). sameday **A 68% vs C 42.5%**. **C(웹)는 앵커당 데이터 3~7배**인데 A와 같은 연애 클러스터를 띄움 → "웹은 오염"이란 전제가 약화(앱도 68% 인세션 편향, 종류만 다름).
- **수정**: affinity를 **C(웹온리) 또는 A∪C로 재산출**(데이터 이미 있음, BQ 0). "앱=깨끗" 주장하려면 *마케팅유입 vs 오가닉* 축을 직접 분리해야(현재 데이터에 없음). 최소 웹·앱 top-N 교집합만 신뢰.

## 우선순위 3 — 지표·공식 정확성

### P7. lift 분모 오류 + self-pair 이중카운트
- **수렴**: 에이전트 1(F6)·2(F4·F8)
- **근거**: base=endpoint 빈도 ≠ NBO prior `P(Y as 2nd)` → 인기 파트너 **1.3~1.5배 오스케일**(43513 lift 2.16 → 실제 ~1.4). self-pair(X→X)가 nX 분모에 2씩 더해져 모든 conf 깎임(2141 **-7.3%**).
- **수정**: lift 분모 = `(s_menu==Y).mean()`(정방향) 또는 PMI/Jaccard 단일 채택. self-pair 분모 제외.

### P8. conf vs lift 혼용 + popularity 지배 (단, 전체순위는 인기 아님)
- **수렴**: 에이전트 2(F5)·3(F7)·4(S6)
- **근거**: 랭킹은 conf인데 보고는 lift 강조, **Kendall tau 0.21**(거의 독립). 속마음(2141)이 **47% 앵커**에 침투(일부 lift<1인데도 rank-1 = 반증분). **단 credit**: 앵커별 순위 vs 글로벌인기 Spearman 0.32 → affinity가 앵커 고유 신호 담음(인기로 환원 안 됨).
- **수정**: 추천 기준 단일화(증분이면 lift), conf는 신뢰필터. 앵커당 글로벌 top-K 1개로 캡 + 다양성 패널티.

### P9. REVISION 3 역행
- **수렴**: 에이전트 2(F9)·4(meta)
- **근거**: 이 산출물이 baseline R3가 고친 두 원칙을 **둘 다 되돌림** — ① sameday≠revisit 분리(여기선 ever-2nd 풀링) ② "연애 디폴트=base-rate 가공물, lift로 봐야"(여기선 conf 정렬).

## 처방 — 가성비순 (대부분 스크립트 수정 + 기존 데이터로 즉시)

1. **하드 게이트 3종** (P1·P2·P10): `is_open & not stop_selling & buyers_30d>0` + 비콘텐츠/업셀 blocklist + 자기·보유스킬 제외. → rank-1의 ~35% 죽은 추천 제거. **6/9 전 최우선.**
2. **방향·표면 분리** (P3·P4): 정방향만 + 결제직후=`s_menu`/재방문푸시=`r_menu` 2벌. 
3. **표본 게이트 코드화** (P5): pair_n≥30·anchor_n2≥50 + Wilson 하한, CSV 단계 강제.
4. **모집단 재산출** (P6): C(웹온리) 또는 A∪C affinity 병기 — 웹 적용이면 C가 데이터 3~7배·동일 클러스터.
5. **공식 정정** (P7·P8·P9): lift 분모=P(Y as 2nd) 또는 PMI/Jaccard, self 제외, lift 정렬 — R3 교정 재적용.
6. **데이터 위생** (P11·P12): NaN 드롭 로깅(ISS), 카탈로그 window 일치 재추출.
7. **운영** (P13·P15·P16): 출시=A/B 게이트, 이탈자 fallback, 다양성 탐색.

**근본 한계**: co-purchase에 **노출(impression) 분모가 없어** "관심"과 "과거 노출"을 분리 불가. 진짜 해법은 노출 로그 기반 lift — 별도 데이터 작업(`/dev-data`).
