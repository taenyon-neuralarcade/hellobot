| # | 발견 | 적용 함의 |
|---|---|---|
| 1 | 추천은 **co-purchase NBO** 가 최강 (적중률 36.6%). 인기순 32.5% · 같은주제 15.2% · 재구매점수순 1.7%(작동X). 주제정보 섞어도 36.6→36.7 | demographic 개인화 말고 **행동 기반 co-purchase 한 방**으로 시작. 콘텐츠(스킬) 단위. cold-start = 인기순 fallback |

→ 적용 = "똑똑한 알고리즘 출시"가 아니라 **노출 표면을 먼저 깔고 노출량을 늘리는 것**이 1순위.

## 1. 공통 엔진 — co-purchase NBO 테이블

모든 표면이 공유하는 단일 입력값.

- **정의**: `(방금 본/산 스킬) → 다음 top-N 스킬` 매트릭스. co-purchase(이어서 산 조합) 기반, **콘텐츠(스킬) 단위**.
- **cold-start**: 판매 데이터 적은 스킬 / 막 유입된 신규 → **전체 인기순 fallback** 으로 빈자리 보강.
- **산출**: BigQuery (Phase 1~4 baseline 자산 재활용) → 서버 API 서빙. (`/dev-data` + `/dev-server`, 구현은 후속)
- **하지 말 것**: 같은주제 좁히기(15.2%), 재구매점수순(1.7%), 성별·나이 세분화 개인화 — 데이터로 효과 작음 확인됨.

## 3. 1순위 리드 상세 — 결제 직후 인세션 NBO 슬롯

**무엇**: 앱에서 스킬 결제 완료 / 결과 확인 직후 화면에 NBO top-3~5 슬롯("이 운세 본 분들이 다음으로 많이 본 운세"). 로직 = §1 공통 엔진, 신규/신상품은 인기순 fallback.

**왜 1순위** (3중 근거):
1. 재구매 낭떠러지는 1→2 한 곳 → 결제 직후가 가장 결정적 순간 (당일 인세션 업셀).
2. co-purchase 가 적중률 36.6% 최강 + 가장 단순.
3. **현재 추천 노출량 자체가 적음** → 이 슬롯이 노출량을 만드는 출발점. 알고리즘 정교화보다 노출 표면 신설이 먼저.

**핵심 메시지 (미팅):** "정교한 개인화보다 노출 표면을 먼저 깔자. 결제 직후 한 슬롯이 가장 임팩트 크고 가장 단순하다."

**성공 측정**:
- 북극성: 첫 구매자 → 2번째 구매 전환율 (당일 + 재방문).
- 슬롯 단위: 노출수 · CTR · 슬롯 경유 2번째 구매 전환.
- **A/B**: 슬롯 노출군 vs 비노출군의 **증분** 2번째 구매율. (적중률 36.6%는 "이미 사려던 사람 맞힘" 수치라, 증분 인과효과는 노출 실험으로만 확정)

**6/9 미팅에서 결정받을 열린 질문**:
- 슬롯 위치/UX — 결제완료 화면 vs 스킬결과 하단 vs 둘 다? (디자인·앱 협의)
- NBO 테이블 갱신 주기 / 서빙 방식 (배치 vs 실시간) — 데이터·서버
- 미끼상품(신년·임신·결혼) 슬롯 내 취급 — 역할 태그 선반영 여부
- A/B 설계 — 분할 비율 · 기간 · 최소 표본

**합의 후 후속 (별도 프로젝트로 분리)**:
- `/dev-data` — co-purchase NBO 매트릭스 산출 (BigQuery)
- `/dev-server` — NBO 서빙 API
- `/dev-ios` + `/dev-android` — 결제 직후 슬롯 UI
- `/design` — 슬롯 디자인 스펙
