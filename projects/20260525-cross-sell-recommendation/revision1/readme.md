## 배경

헬로우봇 앱에서 1회 결제 경험이 있는 사용자가 두 번째 결제로 이어지지 않는 갭이 큼. 이 갭을 데이터로 해부하고, 어떤 사용자 세그먼트에게 어떤 SKU 를 추천해야 두 번째 구매로 전환되는지에 대한 가설·실행 후보를 도출.

## 분석 의도 (사용자 합의 2026-05-25)

> "현재 유입되는 사용자가 어떤 특정 콘텐츠를 구매했을 때 다음 콘텐츠를 어떤 걸 추천/노출 해야 해당 사용자가 관심을 보이고 재구매에 이르게 할 지에 대해 특정한 사용자 세그먼트를 분류하고, 그 세그먼트의 기존 사용자 특성을 분석하여 추천 콘텐츠를 도출"

- 여러 기준으로 사용자 세그먼트를 만들어 보고
- **특성이 명확하게 드러나는 세그먼트부터 우선** 공략
- 세그먼트별 행동 특성을 근거로 추천 SKU 도출

## KPI

| 지표 | 정의 |
|---|---|
| **재구매율** (Primary) | 첫 결제 후 N일 이내 두 번째 결제 발생률. N = 30/60/90 병행. **첫구매 채널(A/B/C) × 신규/기존 분리 측정** |
| **2회차 타이밍 분리** | ① 동일일 2회차 (인세션 업셀) ② 재방문(익일+) 2회차 (리텐션 재구매) — 별도 측정 |
| **두 번째 결제까지 median 일수** | 시간 임계점 파악용 (재방문 케이스 중심) |
| **추천 lift** | (세그먼트 × 첫 SKU) → 두 번째 SKU 의 lift (전체 대비 N배) |
| **세그먼트 커버리지** | 전체 결제자 중 우선 공략 세그먼트 인원·매출 비중 |

### 첫구매 채널 분리 (최상위 분리축)

웹/앱 유입 동기와 사용 방식이 달라 **첫구매 채널을 최상위 분리축**으로 둠. 두 필드로 판별:
- `user_first_paid_date` — 웹/앱 통합 진짜 첫 구매일
- `user_first_app_paid_date` — 앱 한정 첫 구매일

| 그룹 | 판별식 | 정체 | 본 분석 스코프 |
|---|---|---|---|
| **A. 앱-퍼스트** | `user_first_paid_date == user_first_app_paid_date` | 진짜 앱 첫 구매자 | **1순위 (메인)** — 풀 분석 |
| **B. 웹→앱 이동자** | `user_first_paid_date < user_first_app_paid_date` | 웹 먼저 → 앱 이동 (채널이동 = 리텐션 신호) | **2순위** — 별도 코호트 격리, "앱 전환 유발 요인" |
| **C. 웹-온리** | 앱 구매 없음 | 웹 마케팅 1회성 전환 모집단 | **후속 트랙** — 본 분석 제외 (규모만 확인). 마케팅 스킬 선정 분석으로 별도 진행 |

> 채널 통합 집계 금지 — 채널 이동자를 신규 첫구매자로 오분류하고, 두 모집단 평균이 무의미해짐.

## 핵심 가설 (분석으로 검증)

- H1. **재구매 전환의 시간 임계점** — 첫 결제 후 X일 이내 재구매 없으면 churn 가능성 급증
- H2. **카테고리 fit** — 동일/인접 카테고리 추천 fit > 무관 카테고리
- H3. **세그먼트별 차이** — 연령·성별·토픽·가격대 등 축 조합별 재구매 패턴 상이 (의미있는 축을 데이터로 선별)
- H4. **첫 결제 가격대 영향** — 저단가 (예: 하트 충전권) vs 고단가 (스킬 이용권) 사용자 재구매 행태 차이
- H5. **동일일 vs 재방문 2회차의 동인 차이** — 동일일 업셀과 재방문 리텐션 재구매는 다른 시그널·적용 위치(결제직후 추천 vs 푸시·홈탭)로 갈림
- H6. **신규 vs 기존 결제자 동학 차이** — 신규는 허니문 버스트, 기존은 재활성화 — 추천 타이밍·콘텐츠가 달라야 함
- H7. **첫구매 채널별 재구매 동학 차이** — 웹 마케팅 유입 첫구매자는 1회성 이탈 성향 강함. 앱-퍼스트가 가장 깨끗한 인앱 cross-sell 타겟. 웹→앱 이동자는 채널이동 자체가 강한 리텐션 신호
