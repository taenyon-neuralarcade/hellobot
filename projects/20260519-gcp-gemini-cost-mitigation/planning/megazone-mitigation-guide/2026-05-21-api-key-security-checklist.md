# API Key 보안 점검 및 권한 제한 가이드

> 트랙 3 (보안 점검) 의 SSOT 가이드 문서.

안녕하세요, 메가존소프트 김종현입니다.


최근 생성형 AI 서비스의 사용이 증가함과 동시에, 클라이언트(웹 소스코드, 모바일 앱 패키지 등) 환경에 노출된 퍼블릭 API Key를 탈취하여 고과금 AI API를 무단으로 호출하는 글로벌 어뷰징(Abusing) 공격이 발생하고 있습니다.

과거 식별 목적으로 생성되어 클라이언트 환경에 배포된 일부 API Key의 경우, 별도의 제한 설정이 없다면 생성형 AI(Gemini API)를 포함한 고과금 서비스의 호출 권한까지 열려있을 가능성이 있어 재무적 리스크가 발생할 수 있습니다.

이에 고객사의 자산을 보호하고 안전한 클라우드 환경을 유지하기 위한 API Key 보안 상태 점검 및 권한 제한 가이드를 안내해 드립니다. 아래 내용을 확인하시어 즉각적인 보안 조치를 취해주시기 바랍니다.

## 1. 현재 보안 상태 점검 방법 (GCP 콘솔)

프로젝트 내에 무단 도용 위험이 있는 '제한 없는(Unrestricted) API Key'가 존재하는지 먼저 확인해야 합니다.

- Google Cloud 콘솔에 로그인합니다.
- 메인 메뉴에서 [API 및 서비스] > [사용자 인증 정보 (Credentials)]로 이동합니다.
- 'API 키(API Keys)' 목록을 확인합니다.
- 키 이름 옆에 황색 경고 아이콘(⚠️)이 표시되어 있거나, 제한사항 항목이 '없음(None)' 또는 '적용 안 됨'으로 되어 있다면, 해당 키는 해커의 타겟이 될 수 있는 보안 취약 상태입니다.

## 2. 보안 조치 가이드 (고객사 환경별 대응)

보안을 강화하는 가장 확실한 방법은 기존 키를 폐기하고 새로운 키로 교체(Rotate)하는 것이지만, 서비스 환경에 따라 즉시 교체가 불가능한 경우가 있습니다. 고객사의 상황에 맞는 유형을 선택하여 조치해 주십시오.

### 💡 유형 A. API Key 교체(Rotate)가 가능한 경우

**대상**: 백엔드 서버에서만 키를 사용 중이거나, 웹 프론트엔드 배포가 즉시 가능하여 키 변경을 바로 반영할 수 있는 경우

1. **신규 키 생성**: [사용자 인증 정보] 메뉴에서 [사용자 인증 정보 만들기] > [API 키]를 클릭하여 새로운 키를 생성합니다.
2. **제한 설정 조치**: 신규 생성된 키에 대해 아래 '3. API Key 제한 설정 방법'을 참고하여 사용처와 필수 API만 화이트리스트로 지정합니다.
3. **소스코드 반영**: 신규 키를 애플리케이션에 반영하고 정상 작동을 확인합니다.
4. **구형 키 삭제**: 기존에 유출 위험이 있던 구형 API 키를 콘솔에서 완전히 삭제(Delete)합니다.

### 💡 유형 B. 하드코딩 등으로 인해 즉시 키 교체(Rotate)가 불가능한 경우

**대상**: 모바일 앱(Android/iOS) 내에 Firebase API Key가 하드코딩되어 이미 스토어에 배포되었거나, 키 값을 변경할 경우 기존 사용자의 앱이 마비되는 경우

이미 배포된 키 스트링(문자열) 자체를 바꿀 수 없다면, 키 값을 유지한 채 구글 클라우드 백엔드에서 해당 키의 '호출 권한'만 강제로 묶어두어야 합니다. 키를 삭제하거나 바꾸지 않고도 Gemini API 호출만 완벽하게 차단할 수 있습니다.

1. **키 값 유지**: 앱에 하드코딩되어 있는 기존 API Key를 그대로 유지합니다. (삭제 금지)
2. **백엔드 권한 통제**: 구글 콘솔에서 해당 키의 'API 제한사항(API Restrictions)' 기능을 활성화하여, 이 키로는 오직 기존에 사용하던 Firebase 관련 서비스만 호출할 수 있도록 화이트리스트를 지정합니다.
3. **효과**: 이렇게 설정하면 해커가 동일한 API Key를 가지고 있더라도, 백엔드 게이트웨이에서 Gemini API(Generative Language API) 호출 요청을 원천 차단(403 Forbidden)하기 때문에 앱은 정상 작동하면서 이상 과금 리스크는 완벽히 소멸합니다.

## 3. 상세 설정 방법 (유형 B 필수 적용 가이드)

즉시 키 교체가 불가능한 Firebase 사용 고객사 분들은 아래 단계를 따라 기존 키에 대한 API 제한을 반드시 설정해 주십시오.

### [1단계] API 제한사항 메뉴 진입

- GCP 콘솔 [API 및 서비스] > [사용자 인증 정보]로 이동합니다.
- 수정할 기존 API Key의 이름을 클릭하여 편집 화면으로 진입합니다.

### [2단계] API 제한사항(API Restrictions) 설정

- 화면 하단의 'API 제한사항(API Restrictions)' 섹션으로 이동합니다.
- 라디오 버튼을 [키 제한 (Restrict key)]으로 변경합니다.
- 드롭다운 메뉴를 클릭하여, 고객사의 앱이 작동하는 데 필요한 Firebase 전용 API들만 체크(선택)합니다.

#### 📌 Firebase 앱 정상 작동을 위한 필수 권장 API 리스트 (선택 항목)

고객사가 Firebase의 어떤 기능을 쓰느냐에 따라 아래 항목들을 찾아 체크해 주십시오. (앱 작동에 필요한 항목만 선택)

- **Firebase Installations API** (Firebase 기본 구동에 필수)
- **Identity Toolkit API** 또는 **Firebase Authentication API** (사용자 인증/로그인 사용 시)
- **Cloud Firestore API** (Firestore 데이터베이스 사용 시)
- **Firebase Remote Config API** (원격 구성 사용 시)
- **Firebase Cloud Messaging API** (푸시 알림 사용 시)
- **Google Analytics for Firebase API** (앱 분석 사용 시)

### [3단계] Gemini API 차단 및 저장

- 선택 항목 중 **Generative Language API** (또는 **Vertex AI API**) 항목이 절대 체크되지 않았는지 다시 한번 확인합니다. (체크 해제되어 있어야 차단됩니다.)
- 화면 하단의 [저장 (Save)] 버튼을 누릅니다.

> **주의**: 설정이 백엔드 시스템 전체에 반영되기까지 약 5~10분의 시간이 소요될 수 있습니다.

## 4. 추가 예방 조치 권장 사항

API Key 자체의 보안 조치 외에도, 만에 하나 발생할 수 있는 이상 과금을 조기에 감지하기 위해 아래 설정을 추가로 적용하시는 것을 강력히 권장합니다.

- **결제 알림(Billing Alerts) 설정**: [결제(Billing)] > [예산 및 알림] 메뉴에서 일일 또는 월간 예상 비용이 기준치(예: 평소 비용의 120%)를 초과할 경우 담당자들에게 즉시 이메일 알람이 발송되도록 예산을 설정하십시오.
- **서비스별 할당량(Quota) 제한**: 당장 Gemini API를 사용할 계획이 없다면, [API 및 서비스] > [사용 설정된 API 및 서비스]에서 Generative Language API를 찾아 선택한 뒤, [할당량] 탭에서 일일 최대 호출 가능한 분량(Quota)을 0 또는 최소치로 줄여두시면 완벽한 이중 방어선이 구축됩니다.

본 가이드와 관련하여 기술적인 지원이 필요하시거나 설정에 어려움이 있으신 경우, 언제든지 메가존소프트 서포트 포털 또는 담당 영업 대표를 통해 기술 지원을 요청해 주시기 바랍니다.

고객사의 안전한 클라우드 운영을 위해 항상 최선을 다하겠습니다.
