| 영역 | 영향 | 비고 |

| 서버 (hellobot-server) | 푸시 발송 시 앱 버전별 분기 → 신/구 Braze API 키 양쪽으로 발송 | 이전 완료까지 듀얼 발송 유지 |
| Braze 콘솔/계정 | 신 계약 측 워크스페이스·API 키·SDK 키 발급 | 계약 이전 절차 |
| 운영 (마케팅·CRM) | 캠페인·세그먼트·캔버스 마이그레이션 | Braze 측 마이그레이션 도구 또는 export/import |

### 제약·고려사항

- **이전 후 미업데이트 사용자**는 더이상 푸시 수신 불가 → 사용자 손실 최소화가 핵심
- 1달 듀얼 푸시 운영 기간 동안:
  - 신 키 = 업데이트한 사용자 도달
  - 구 키 = 미업데이트 사용자 도달 (구 계약 만료 직전까지)
- 중복 푸시 방지: 동일 사용자가 신/구 양쪽 동시 수신 가능성 → 앱 버전 분기로 한쪽만 발송
- 마케팅 자동화 캠페인이 진행 중이면 캠페인 단위 이중화도 필요
- 푸시 토큰(APNs/FCM)은 신 워크스페이스에서 재수집되어야 함 → 신 빌드 첫 실행 시 신 워크스페이스에 등록
